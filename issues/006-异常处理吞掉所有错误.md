# 006 多处使用裸 except 吞掉所有异常

**严重程度**: MEDIUM  
**文件**: `data/fetch.py:46,69`, `sector.py:54,83`, `run_daily.py:132`, `models/factors.py:357`  
**类型**: 代码质量

## 问题描述

项目中大量使用 `except Exception: pass` 或 `except Exception as e: print(...)` 后继续执行：

| 位置 | 代码 |
|------|------|
| `data/fetch.py:46` | `except Exception as e: print(...); return pd.DataFrame()` |
| `data/fetch.py:69` | `except Exception as e: print(...); return pd.DataFrame()` |
| `sector.py:54` | `except Exception: pass` |
| `sector.py:83` | `except Exception: pass` |
| `run_daily.py:132` | `except Exception: pass` |
| `models/factors.py:357` | `except Exception: pass` |

这会导致：
- 数据获取失败时静默跳过，用户不知道哪些股票漏掉了
- 因子计算失败时没有任何记录
- 排查问题极其困难

## 修复建议

1. 引入 logging 模块，区分 WARNING 和 ERROR
2. 记录失败的具体原因和上下文（symbol, date, traceback）
3. 统计失败率，超过阈值时报错退出
