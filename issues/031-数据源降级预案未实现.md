# 031 — 数据源降级预案未实现

**严重程度**：MEDIUM  
**审查日期**：2026-05-02  
**状态**：待实现  
**来源**：设计文档 V2.5

## 问题描述

整个系统强依赖 akshare，无备选数据源，无降级策略。akshare 作为爬虫稳定性无法保证，一旦接口不可用系统完全停摆。

## 应实现规则

```
L0（正常）: akshare 全量 → 全部功能
L1（部分接口不可用）: 资金面降级 L3/L4，其余正常
L2（核心接口不可用）: 尝试 baostock/tushare/缓存
L3（所有数据源不可用）: 纯缓存出报告，不生成新信号
L4（缓存过期>3天）: 停止推送，发告警
```

## 备选数据源

| 数据源 | 覆盖 | Token | 优先级 |
|--------|------|-------|--------|
| akshare | 全量 | 无 | 默认 |
| baostock | 个股日线 | 无 | 先实现 |
| tushare | 行情+指数+资金流 | 需要 | 后补 |

## 降级检测

自动：连续3次请求失败→标记不可用，每30分钟尝试恢复，连续2次成功→恢复上一级

## 影响模块

`data/interfaces.py`（已有占位，需补实现），`data/sources/akshare.py`, `data/sources/baostock.py`
