# 019 — 分类指数数据加载无并发、无缓存

**严重程度**：HIGH  
**审查日期**：2026-05-02  
**状态**：待修复  
**文件**：`models/sector.py:36-86`

## 问题描述

`fetch_sector_indices()` 有 28 个申万行业指数 + `fetch_broad_indices()` 有 6 个宽基指数 = **34 个顺序 HTTP 请求**，每次 `load_data()` 至少 34 秒。且每次运行都重新拉取全部 7 年历史数据。

## 影响

- `run_daily.py` 每次启动都要等 30-40 秒加载指数数据
- 历史日线数据不会变，每天重复拉取浪费带宽和时间
- 网络稍微不稳定，部分指数获取失败，分数计算不完整

## 修复方案

1. **并发请求**：使用 `concurrent.futures.ThreadPoolExecutor(8)` ，34 个请求约 4-5 秒完成
2. **本地缓存**：首次拉取后存为 `data/raw/sector_indices.parquet`，后续运行只增量更新最近 N 个交易日
3. **缓存检查逻辑**：检查缓存文件日期，如果最新日期 < 今天-1 天则增量更新

```python
from concurrent.futures import ThreadPoolExecutor, as_completed

def fetch_sector_indices(start, end):
    result = {}
    with ThreadPoolExecutor(max_workers=8) as executor:
        futures = {
            executor.submit(_fetch_one_sector, code, name, start, end): code
            for code, name in SW_INDEX_MAP.items()
        }
        for future in as_completed(futures):
            code, df = future.result()
            if df is not None:
                result[code] = df
    return result
```
