## 核心发现总结

### 关于"分类指数因子"的三个关键断层

1. **技术分析断层**：个股有四层递进筛子 (L1趋势 → L2结构/6类形态 → L3时机/MACD → L4共振/周线)，分类指数只有初级 MA排列+动量，缺少年线过滤、形态识别、MACD、周线确认等全部核心检查。

2. **回测集成断层**：`BacktestEngine.run()` 调用 `score_single()`（不含L0），从未调用 `score_with_sector()`（含L0），且未初始化 sector engine。整个 L0 分类指数加权在回测中完全未生效，只在日报 PDF 中展示。

3. **分类映射断层**：`assign_stock_sectors()` 按代码前缀做4类粗映射（60/00→沪深300+中证500, 300→创业板, 688→科创50），完全没有行业归属，且缺少 ETF 分类指数体系。

### 关于回测可信度

**013号bug（换手率恒为0）** 是所有回测相关bug中最严重的——从第二个调仓期开始交易成本从未扣除，回测收益被系统性高估。修复后需要重新跑全部回测验证。

1. **资金面角色**: 从"门卫AND条件"改为"确认器"——缺失≠否定，只有反向流出才不通过
2. **动态阈值策略**: 冷启动用排序（前30%），不写死数值；积累数据后回测反推

5. **Alpha基准**: 沪深300（统一基准，不用评估对象自身），回测后验证
6. **缠论分值**: 所有分值为冷启动占位值，纳入回测校准周期，不纠结具体数字

## 第五批 — 2026-05-03 回测 + 自动调参

完整报告: `issues/BACKTEST-2026-05-03.md`  
数据范围: 2019-01 ~ 2026-04, 88 个调仓日, 30 只真实股票

### 最佳参数

| 参数 | 值 |
|------|-----|
| top_n | **20** |
| weighting | **equal** |
| pipe_weights | **T:0.60, C:0.25, F:0.15** |

### 最佳回测指标

| 指标 | 值 |
|------|-----|
| 年化收益 | **+11.11%** |
| 累计收益 | +87.55% |
| Sharpe | **0.407** |
| 最大回撤 | 32.67% |
| 胜率 | 54.3% |

### 关键发现

1. **等权 > 评分加权**：评分加权(6.67% ann)反而比等权(10.55%)低近4%，因子区分度不足
2. **top_n=20 最优**：太少(5)收益低回撤大，太多(30+)稀释Alpha
3. **技术面权重最高**：T:60% > T:55% > T:50%，基本面对收益贡献有限
4. **月频对成本不敏感**：cost 0→0.3%仅降年化1.13%
