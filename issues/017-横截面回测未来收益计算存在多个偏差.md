**状态**：待修复  
**文件**：`backtest_cross_section.py:148-171`

## 问题1：停牌导致持有期不一致

```python
# line 159-166
if len(next_me) == 0:
    close_next = df_after['close'].iloc[-1]  # 用最近可用收盘价
```

当股票停牌跨越月末时，用恢复交易后的第一个收盘价。但检查条件是 `(df_after.index[-1] - pd.Timestamp(me_str)).days > 40`，以月初而非下月末为基准。

**举例**：me_str=3月31日，股票3月28日停牌，5月10日复牌。时间差=40天，不会跳过。实际持有期=40天，而正常股票持有期=28天。不同股票持有期不一致，收益不可比。停牌复牌后的跳空被算入"因子选股能力"，实际是停牌补偿。

## 问题2：未显式指定前复权

```python
# line 67
df = ak.stock_zh_a_hist_tx(symbol=code, start_date=..., end_date=...)
```

未传 `adjust` 参数。需要确认 akshare 默认行为并显式传入 `adjust="qfq"`（前复权），否则股息/拆股会导致价格断裂。

## 问题3：me_str 可能不是真实交易日

只检查 `me_str in df.index`（line 156），不检查 me_str 离真实最后交易日有多远。如果某股票实际最后交易日是3月28日而me_str是3月31日（东方财富填充了非交易日），close_now 可能是填充值而非真实收盘价。

## 修复方案

1. 显式传入 `adjust="qfq"` 前复权参数
2. 持有期收益计算统一到下一个 rebalance 日（下月月末），停牌股票直接排除
3. 停牌偏差超过 5 个交易日的样本从 IC 计算中排除
4. 检查 me_str 是否在 -2 天内接近真实最后一个交易日
