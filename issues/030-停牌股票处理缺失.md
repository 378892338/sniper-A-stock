# 030 — 停牌股票处理机制缺失

**严重程度**：MEDIUM  
**审查日期**：2026-05-02  
**状态**：待实现  
**来源**：设计文档 V2.5

## 问题描述

当前系统对停牌股票没有任何特殊处理。持仓股票突然停牌时，系统可能将其计入正常持仓、错误地尝试卖出操作，或者因获取不到价格而崩溃。

## 应实现规则

```
停牌期间:
├─ 状态标记为 FROZEN
├─ 不计入可操作持仓权重
├─ 仓位计算: 总仓位 = 可操作仓 + 冻结仓
└─ 冻结仓 > 50%总仓位 → 告警，暂停新买入

复牌后:
├─ 首日不操作，观察
├─ 次日重新评估
└─ 停牌期间错过退出信号 → 复牌后补检，触发则直接卖出
```

## 影响模块

`engine/state_machine.py`, `engine/watcher.py`, `engine/portfolio.py`
