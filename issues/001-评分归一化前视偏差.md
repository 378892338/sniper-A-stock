# 001 评分归一化存在前视偏差 (Look-ahead Bias)

**严重程度**: CRITICAL  
**文件**: `models/factors.py:406-418`  
**类型**: 回测作弊 / 数据泄漏

## 问题描述

`calc_all_factors` 中三个管道的评分使用了整个时间序列的最大值做归一化：

```python
tech_score = tech_score / tech_score.max() * 100  # line 406
cap_score = cap_score / cap_score.max() * 100      # line 413
fund_score = fund_score / fund_score.max() * 100   # line 419
```

这意味着 2019 年的评分使用了 2026 年的最大值来缩放。回测中早期日期的得分被人为压低或抬高，导致回测结果不可信。

## 修复建议

改用滚动窗口归一化或截面排名：

```python
# 方案A: 滚动窗口 (比如 252 天)
tech_score = tech_score / tech_score.rolling(252).max() * 100

# 方案B: 截面百分位排名 (推荐，配合 sieve.py 的 rank_series)
# 不在 calc_all_factors 里做归一化，统一交给外层的 rank_series 处理
```

## 影响范围

- `backtest/run.py` — 回测结果虚高
- `backtest_cross_section.py` — IC分析、分层回测结果虚高
- `run_daily.py` — 日报评分有偏
