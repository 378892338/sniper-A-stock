# 016 — 回测引擎未集成 L0 分类指数加权

**严重程度**：HIGH  
**审查日期**：2026-05-02  
**状态**：待修复  
**文件**：`backtest/run.py:144`, `strategies/sieve.py:52-74`

## 问题描述

`BacktestEngine.run()` 调用了 `score_single()` 而非 `score_with_sector()`，且从未初始化分类指数引擎。整个 L0 层在回测中完全未生效。

## 证据

```python
# backtest/run.py:144
s = strategy.score_single(df.loc[:rd], wdf, symbol=sym)  # ← score_single 不包含 L0
```

`sieve.py:42-43` 的文档已写明：
```python
def score_single(...):
    """单只股票因子评分（不含L0分类指数加权）。"""
```

而包含 L0 加权的方法 `score_with_sector()` 已经实现（`sieve.py:52-74`），但回测引擎从未调用它。

另外，`BacktestEngine.run()` 也没有调用 `strategy.init_sectors()`，即使调用 `score_with_sector()`，`sector_engine` 也是空的。

## 影响

- 所有回测结果（IC、分层收益、夏普等）都是**纯因子打分**，L0 分类指数加成完全未被纳入
- `sector.py` 的代码只在 `run_daily.py` 日报生成中使用，回测中形同虚设
- 用户看到的回测绩效和策略设计的理论绩效不一致

## 修复方案

1. `BacktestEngine.run()` 中改用 `score_with_sector()` 替代 `score_single()`
2. 在 `run()` 开始时调用 `strategy.init_sectors(start, end)` 初始化分类指数数据
3. 或者在 `BacktestEngine.__init__()` 中接受一个 `init_sectors` 参数
