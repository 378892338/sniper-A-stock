## 2. 调仓日期类型不兼容风险

**文件**：`backtest/run.py:18,142`

```python
dates = [d.strftime("%Y-%m-%d") for d in dates]  # 转为字符串
# ...
if rd in df.index:  # df.index 是 DatetimeIndex
```

pandas 的 `in` 运算符支持字符串到 Timestamp 的隐式转换，但这种隐式行为在 pandas 版本变更时可能出问题。建议使用 `pd.Timestamp(rd) in df.index` 显式转换。

## 3. run_daily.py 和 backtest_cross_section.py 分组逻辑不一致

## 4. O(n²) 时间复杂度的线性搜索

**文件**：`backtest/run.py:168`

```python
next_date = rebalance_dates[rebalance_dates.index(rd) + 1]
```

`rebalance_dates.index(rd)` 在列表中做线性搜索。对月度调仓（约84次）影响可忽略，但日频调仓（约1800次）总复杂度为 O(n²)。建议用 enumerate：

```python
for i, rd in enumerate(rebalance_dates):
    next_date = rebalance_dates[i + 1] if i + 1 < len(rebalance_dates) else self.end
```
