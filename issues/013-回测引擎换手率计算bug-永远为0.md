# 013 — 回测引擎换手率计算 bug：turnover 恒为 0

**严重程度**：CRITICAL  
**审查日期**：2026-05-02  
**状态**：待修复  
**文件**：`backtest/run.py:136-163`

## 问题描述

`BacktestEngine.run()` 中 `prev_holdings = holdings`（第151行）的位置错误，导致从第二个调仓期开始换手率恒为 0。

## 根因分析

```python
# line 136-137
prev_holdings = []

for rd in rebalance_dates:
    holdings = strategy.select_top(scores, rd, self.top_n)
    if len(holdings) < 5:
        holdings = prev_holdings
    prev_holdings = holdings   # ← 第151行：这里更新了 prev_holdings

    # ...

    if prev_holdings:           # ← 第158行：prev_holdings == holdings
        sold = set(prev_holdings) - set(holdings)   # 永远是空集
        bought = set(holdings) - set(prev_holdings)  # 永远是空集
        turnover = 0  # ← 恒为 0
```

第一次迭代 `prev_holdings` 是空列表 `[]`，走 `else` 分支 → `turnover = 1.0`（正确）。第二次迭代起，`prev_holdings` 在第151行被更新为当前 `holdings`，到第158-160行使用时两者是同一个对象，差集永远为空 → `turnover = 0`。

## 影响范围

- **交易成本从第二个调仓期开始从未扣除**（因为 `turnover_cost = 0 * 0.0015 = 0`）
- 约84次月度调仓中，83次的交易成本被跳过
- 按每次调仓30-50%换手率、千1.5成本计算，累计高估收益约 3-5 个百分点

## 修复方案

将 `prev_holdings = holdings` 移到换手率计算**之后**：

```python
for rd in rebalance_dates:
    holdings = strategy.select_top(scores, rd, self.top_n)
    if len(holdings) < 5:
        holdings = prev_holdings

    # 先用旧的 prev_holdings 计算换手率
    if prev_holdings:
        sold = set(prev_holdings) - set(holdings)
        bought = set(holdings) - set(prev_holdings)
        turnover = (len(sold) + len(bought)) / len(prev_holdings) / 2
    else:
        turnover = 1.0
    turnover_cost = turnover * self.transaction_cost

    # 然后再更新 prev_holdings 供下一轮使用
    prev_holdings = holdings
```
