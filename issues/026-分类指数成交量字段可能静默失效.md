# 026 — 分类指数成交量字段可能静默失效

**严重程度**：MEDIUM  
**审查日期**：2026-05-02  
**状态**：待修复  
**文件**：`models/sector.py:53`, `models/sector.py:80`

## 问题描述

```python
# sector.py:53 (fetch_sector_indices)
df['volume'] = df.get('amount', 0) / df['close'] * 100

# sector.py:80 (fetch_broad_indices)
df['volume'] = df.get('amount', 0) / df['close'] * 100
```

两个函数都用了 `df.get('amount', 0)` — 如果 akshare 返回的 DataFrame 中 `amount` 字段不存在（例如 API 变更），`get` 返回默认值 0，导致 `volume` 全为 0。

此时 `calc_sector_strength()` 中第 115-119 行的量能趋势因子静默失效：

```python
if 'volume' in df.columns:         # volume 列存在
    vol_ma = df['volume'].rolling(20).mean()
    vol_ratio = df['volume'] / vol_ma  # 0/0 = NaN
```

不会报错，量能因子静默归零，用户完全不知道。

另外，`* 100` 没有明确意义 — 对于 stock index，`amount` 是成交额（元），除以 close 得到成交量（股），乘以 100 没有必要。

## 修复方案

1. 显式检查 `amount` 字段是否存在，不存在则抛警告
2. 去掉无意义的 `* 100`
3. 如 `amount` 不存在，尝试通过其他方式估算成交量

```python
if 'amount' not in df.columns:
    logger.warning(f"指数 {code}({name}) 缺少成交额字段，量能因子将不可用")
    df['volume'] = np.nan
else:
    df['volume'] = df['amount'] / df['close']
```
