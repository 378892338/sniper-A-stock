# 014 — 异常处理静默吞没关键错误

**严重程度**：CRITICAL  
**审查日期**：2026-05-02  
**状态**：待修复  
**涉及文件**：`run_daily.py`, `backtest_cross_section.py`, `models/factors.py`

## 问题描述

多处使用 `except Exception: pass` 或 `logger.debug()` 记录关键错误，生产环境（日志级别 INFO）下完全不可见。

## 具体位置

### 1. run_daily.py:142-143 — 最严重

```python
except Exception:
    pass
```

每日行情获取失败时静默跳过。如果某天采样15只股票中8只获取失败，最终报告只剩7只，用户完全不知道数据缺失。

### 2. backtest_cross_section.py:126-127

```python
except Exception:
    pass
```

因子计算异常被吞没。某只股票某月因数据问题计算失败，该月该股票从排名中消失且无记录。连续发生会导致某些类型股票系统性丢失，回测结果失真。

### 3. models/factors.py:309-310 — calc_resonance_score

```python
except Exception:
    pass
```

日期映射失败静默跳过，共振分数默认为0，无任何日志。

### 4. data/fetch.py + models/sector.py — logger.debug() 在生产中不可见

所有 `except Exception as e: logger.debug(...)` 在 logger 默认级别 INFO 时完全不输出。用户看不到数据获取失败。

## 修复方案

1. 所有 `except Exception: pass` 改为：
```python
except Exception as e:
    logger.warning(f"处理失败: {e}", exc_info=True)
```

2. data/fetch.py 和 models/sector.py 中所有 `logger.debug` 改为 `logger.warning`

3. models/financials.py:25-26 `except Exception: return {}` 同样改为 warning 级别
