# 三层统一评分架构设计

## 设计背景

当前三层漏斗系统存在以下脱节问题：
- L1 `assess_market()` 在 `scripts/run_strategy.py` 中完全不参与选股
- L2 评分粗糙：趋势用 DIF 斜率+柱强度，Alpha 用简单超额收益，成交量用 ±3 累加无时间衰减，资金 L4 统一 16/20 分
- L3 的 `etf_tags` 从未被填充，`enable_etf_bonus=False` 硬编码关闭
- 28 个申万行业引擎（`models/sector.py`）独立运行，未与 Gate 系统联动
- 成交量模块无趋势感知、无时间衰减、无涨跌量比
- 资金面模块 L2/L3 分支缺失中间层级处理

## 目标架构

```
L1: 3个市场指数 → Gate(2-of-3) → Score(Trend40+Volume30+Fund30) → 仓位
L2: 13只ETF, Gate(2-of-5) → Score(Trend30+Alpha25+Volume25+Fund20)
    辅助: 28个申万行业强度 → 交叉验证 → 强势行业池
L3: 强势池内个股, Gate(2-of-4) → Score(Trend30+Alpha25+Volume25+Fund20)
     例外: 1只独立强势个股名额 → Top 15
```

## 统一评分维度

各层共用四维度框架，但权重不同：

| 维度 | L1 权重 | L2 权重 | L3 权重 | 说明 |
|------|---------|---------|---------|------|
| Trend（趋势） | 40 | 30 | 30 | 5维MACD评估 + 缠论中枢位置 |
| Alpha（超额收益） | - | 25 | 25 | 横截面百分位排名（L1无此维度） |
| Volume（成交量） | 30 | 25 | 25 | 三部分法：趋势感知量价+量趋势+涨跌量比 |
| Fund（资金面） | 30 | 20 | 20 | 直接信号 × 数据权重 + 技术推断 × (1-权重) |

## 趋势强度（5 维 MACD 评估）

| 子维度 | 满分 | 计算方法 |
|--------|------|----------|
| 方向 | 8 | DIF>0向上=8, DIF>0向下=5, DIF<0向上=4, DIF<0向下=1 |
| 力度 | 7 | 当前MACD柱 / 历史最大柱(20周)，截断[0,1] × 7 |
| 加速度 | 5 | DIF二阶导数（斜率变化），加速向上=5，减速向下=1 |
| 稳定性 | 5 | 金叉后持续周数 >13=5, >8=4, >5=3, >2=2, 刚金叉=1 |
| MA结构 | 5 | MA5>MA20>MA60=5, MA5>MA20=3, MA20>MA60=2, 其余=1 |

## Alpha 强度（横截面百分位排名）

- 在 13 只 ETF 中做多周期排名（1周/4周/13周）
- 权重：1周=0.2, 4周=0.4, 13周=0.4
- 得分 = avg(各周期百分位 × 权重) × 25
- 边际变化加成：4周排名比1周排名上升 5%+ → +2 分

## 成交量健康度（三部分法）

| 部分 | 满分 | 方法 |
|------|------|------|
| 趋势感知量价关系 | 40 | 上涨趋势中放量上涨加分；下跌趋势中缩量下跌加分；带指数时间衰减 0.92^t |
| 成交量趋势方向 | 30 | 成交量 MA(5) vs MA(20)，量能持续放大加分 |
| 涨跌量比 | 30 | 上涨日平均量 / 下跌日平均量，量比 >1.2 加分 |

## 资金力度（双路径）

```
fund_score = 直接信号分 × 数据可用权重 + 技术推断分 × (1 - 数据可用权重)
```

### 数据层级与置信度

| 层级 | 条件 | 置信度 |
|------|------|--------|
| L1 | 北向+融资+大单 三者齐全 | 1.00 |
| L2 | 北向+大单（无融资） | 0.85 |
| L2 | 仅北向 | 0.80 |
| L3 | 仅大单 | 0.75 |
| L3 | 仅融资 | 0.75 |
| L4 | 无资金数据 → 技术推断 | 0.60-0.75 |

技术推断：若量价健康（volume_health > 60）+ 趋势向上（DIF > DEA）→ 推断资金流入，给 12/20；否则 4-8/20。

## Gate 看空信号加权

| 信号 | 权重 |
|------|------|
| 周线顶背驰 | ×3 |
| 周线死叉 | ×2 |
| 日线死叉 | ×1 |
| 日线 DIF < 0 | ×1 |

加权总分 ≥ 4 → 拦截（不通过 Gate）。

## L1 联动阈值

| 市场状态 | L2 Gate 要求 |
|----------|-------------|
| 牛市 (3/3) | 3-of-5 |
| 震荡 (2-3/3) | 2-of-5 |
| 偏弱 (2/3) | 2-of-5 |
| 熊市 (0-1/3) | 跳过 L2，直接空仓 |

## 28 个申万行业辅助层

- 仅作为交叉验证信号，不修改任何评分公式
- 对比 L2 Gate 评分排名 vs 申万行业引擎强度排名
- 收敛（方向一致）→ 日报标注 "✓ 确认"
- 背离（方向相反）→ 日报标注 "⚠ 背离，建议观望"
- 按 `config/settings.py` 中 `SECTOR_INTEGRATION.enable_aux_cross_validation` 开关

## 独立强势股例外

- 即使不在强势行业池，若 L3 评分 > 75 且 缠论买点满分（25分），可入选
- 名额由 `SECTOR_INTEGRATION.independent_stock_slots` 控制，默认 1
