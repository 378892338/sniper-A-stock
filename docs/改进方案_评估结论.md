# 三层漏斗改进方案 — 综合评估结论

> 评估范围：`docs/改进方案_回测分析报告.md` 全部 14 项改进 + "三项改造"建议
> 评估团队：3 个独立评估组
> **核心判定：架构正确，因子设计有误**

---

## 零、根因重定位

**三层漏斗架构是正确的**：盯大盘判断仓位 → 选强势板块 → 选强势股，逻辑链条清晰。

回测跑输基准的根本原因不在架构，而在**每层的因子/样本设计**存在方向性错误：

| 层级 | 因子设计错误 | 后果 |
|------|-------------|------|
| L1 大盘 | 月线DIF<0作看空信号、创业板一票否决、拦截阈值过低 | 牛市判偏弱，仓位被压 |
| L2 板块 | 门控阈值方向反了（牛市最严、偏弱最松）、B3底背驰在牛市中不可能出现 | 牛市选不出板块 |
| L3 个股 | 看空阈值固定不区分市场、无评分门槛、无止损 | 边缘股被买入、亏损股死扛 |

**"三项改造"建议与此判断一致**：多因子联合判定 + 动态阈值 + 分状态仓位管理，都是在因子层面修复，而非改架构。

---

## 一、"三项改造"建议评估

外部建议提出"优先实施多因子联合判定+动态阈值+分状态仓位管理三项改造，预计可将牛市识别率从 9.8% 提升至 30%以上"。

### 结论：建议方向正确，但需补充两个前置因子修复

| 三项改造 | 对应因子修复项 | 说明 |
|---------|-------------|------|
| 多因子联合判定 | P0.5 GATE_THRESHOLD 反转 + P0.6 BEARISH_THRESHOLD 反转 + P1.7 L3 动态阈值 | 修正L2/L3因子方向 |
| 动态阈值 | P0.1 拦截阈值 + P0.6 分状态阈值 + P1.7 L3 分状态阈值 | 因子阈值不再一刀切 |
| 分状态仓位管理 | P0.3 仓位映射 | 仓位与市场状态匹配 |

**需要补充的两个前置修复**：

1. **P0.2 月线 DIF 因子修正**：`DIF<0` 不应直接作为看空信号，应改为 `DIF<0 且正在下降`。这是 L1 层因子选错了——牛市恢复期 DIF 从深负区回升到零轴需要 6-12 个月，这段时间 DIF<0 但趋势向上实际上是看多信号。

2. **P0.4 创业板否决因子分级**：创业板在牛市中正常回调的技术信号不应压制全市场评级。这是因子权重分配问题——单一市场的技术指标权重过高。

## 三、为什么 P0+P1 后只能到 7-9%？

因为 P0+P1 修复了**最严重的因子错误**（方向反了的、逻辑错的、缺失的），但还存在**因子的精细度不足**问题：

1. **L2 板块选择仍为二元门控（通过/不通过）**，而非评分排序。50 个板块中即使选了前 5，也只用"通过/不通过"的二元信息，丢失了板块间强弱排序信息。

2. **个股等权持仓**，评分 90 和 50 的股票仓位相同。因子已经给出了评分差异，但持仓没有利用这个差异。

3. **L2/L3 周频运行**，强势板块和个股的切换滞后 1-2 周。

4. **无趋势延续因子**，每周从零开始评估，牛市中已建立的强势不产生任何惯性加分。

## 四、突破 10%+ 的因子改进路径

| 阶段 | 改进 | 因子层面 | 预期收益贡献 |
|------|------|---------|------------|
| P0+P1 | 修正方向错误+补缺失因子 | 因子纠错 | 4%→7-9% |
| **F1** | **板块评分排序替代二元门控** | L2因子精细化 | +1-2% |
| **F2** | **个股置信度加权持仓** | L3因子利用率 | +1-2% |
| **F3** | **震荡市板块轮动子策略** | L2+L3联动 | +1-2% |
| **F4** | **趋势延续加分因子** | L1跨期因子 | +0.5-1% |

### F1：L2 板块评分排序

**现状**（`runner.py:714-715`）：`l2_ok = l2.passed`，二元通过/不通过。`strong_sectors` 列表内的板块等权，区分不出哪个更强。

**改进**：L2 返回板块评分（0-100），买入时按板块评分加权——强势板块的股票获得更高权重，边缘板块即使通过也低权重。

改动范围：`gate/layer2_sector.py` 返回评分字段 + `runner.py` 买入逻辑引用板块评分。

### F2：个股置信度加权

**现状**（`runner.py:789-790`）：`day_ret = np.mean(rets) * filled_ratio * pos_pct`，所有持仓等权。

**改进**：`day_ret = sum(weight_i * ret_i for each stock) * pos_pct`，其中 `weight_i = score_i / sum(all_scores_of_holdings) * (len(holdings)/top_n)`

改动范围：`runner.py` 持仓管理，买入时记录评分，MTM 计算加权。

### F3：震荡市板块轮动

**现状**：L1=震荡（52.7%时间）时，被动持有 50%仓位，不主动操作。

**改进**：震荡市中，每 2-4 周将持仓向当前最强 2-3 个板块集中。利用 A 股震荡市板块分化特征获取超额收益。

改动范围：`backtest/runner.py` 新增震荡市轮动逻辑，或在 `gate/layer2_sector.py` 中增加轮动信号。

### F4：趋势延续加分

**现状**：每周 L1/L2/L3 全部重新评估，无历史状态记忆。

**改进**：L1 连续 2+ 周判牛市 → 仓位加成 +10%；连续 2+ 周该股通过 L3 → 评分加成 +5%。

改动范围：`backtest/runner.py` 状态变量扩展，追踪连续周数。

## 五、最终推荐实施路线

```
第一阶段（因子纠错）：P0.2 → P0.4 → P0.5 → P0.6 → P0.3
                          ↓
第二阶段（补缺失因子）：P1.7 → P1.8 → P1.9 → P1.10 → P1.11
                          ↓
第三阶段（因子精细化）：F1 板块评分 → F2 加权持仓 → F3 震荡轮动 → F4 趋势延续
                          ↓
第四阶段（回测验证）：全量回测 + 逐年分析
```

## 六、风险评估

1. **F1 板块评分排序**：风险低，只是把二元信息替换为连续信息，不会新增负面暴露。
2. **F2 加权持仓**：风险低，集中度略有提升但有限（每只权重变化在 ±5% 以内）。
3. **F3 震荡轮动**：中等风险，轮动交易频率增加带来交易成本上升。需校准换仓周期（2 周 vs 4 周）。
4. **F4 趋势延续**：低风险，本质是一个小幅动量因子。
