# 量化交易系统 — 项目总结

> 生成日期: 2026-06-06

---

## 一、项目概览

**A 股量化选股交易系统**，基于四层狙击手架构（L0-L4），全市场选股 + 板块轮动 + 个股评分 + 动态风控。

### 系统定位

| 维度 | 说明 |
|------|------|
| 交易标的 | A 股全市场（沪/深/创业板） |
| 选股逻辑 | 自顶向下四层漏斗（市场→板块→个股→入场） |
| 持仓风格 | 最多 5 只，分散持仓，动态仓位 |
| 调仓频率 | 每日 |
| 风控风格 | 两层止损（-3% / 回撤 -5%）+ 组合回撤 5% 强制减仓 |
| 数据源 | 东方财富、新浪、同花顺、akshare（自动降级） |

### 当前配置：Strategy A（2026-06-06 起永久切换）

| 参数 | 值 |
|------|:--:|
| `bullish_threshold` | **64**（L0≥64 开仓） |
| 板块选取 | **统一 Top 3**（不再分层） |
| 强势仓位（L0≥70） | **50%** 总暴露，最多 5 只 |
| 偏强仓位（64≤L0<70） | **30%** 总暴露 |
| L4 退出 | 三层：L0≥70 减半仓 / 64-70 减到 30% / <64 全平 |
| 止损 | -3% 初始止损 / -5% 动态止盈 |
| 持仓上限 | 43 天 |

## 二、核心架构：四层狙击手

```
  L0: 市场评分 ──→ 4 维合成（趋势40% + 量能30% + 宽度20% + 北向10%）
   |              输出: 0-100 综合评分，决定是否开仓 + 仓位大小
   |
  L1: 板块排名 ──→ 4 维合成（动量35% + 资金25% + 广度20% + 热度20%）
   |              输出: Top 3 最强板块
   |
  L2: 个股评分 ──→ 12 因子（趋势+量能+MACD+RSI+资金流+大单+
   |               龙虎榜+EPS+ROE+营收增速+市值+换手率）
   |              输出: Top 10 评分个股
   |
  L3: 入场过滤 ──→ 硬条件（价格3-300 / 成交额>100万 / 换手<30%
   |               / 非涨停）+ 软条件（评分≥79 / 板块Top 3）
   |              输出: 最终候选股
   |
  L4: 退出链  ──→ -3% 止损 / -5% 动态止盈 / MA20 跌破 / 超时
                 三层风控（按 L0 分档）
```

关键文件: `sniper/layers/l0_market.py` → `l1_sector.py` → `l2_stock.py` → `l3_entry.py` → `l4_exit.py`

## 四、数据流架构

```
  ┌─────────────┐    ┌──────────────┐    ┌─────────────┐
  │ 外部数据源   │───→│ SignalStore   │───→│ 本地数据仓库 │
  │ 东方财富     │    │ (归一化+校验)  │    │ (meta.db    │
  │ 新浪财经     │    │               │    │  + parquet) │
  │ 同花顺       │    │               │    │             │
  │ akshare      │    │               │    │             │
  └─────────────┘    └──────────────┘    └─────────────┘
                           │
                           ▼
                    ┌──────────────┐
                    │ L2 缓存      │
                    │ (预计算因子)  │
                    └──────────────┘
```

数据源自动降级: `DATA_SOURCE_PREFERENCE` 配置，主源失败自动切换备源。

## 五、打字机归因系统（在线自进化）

```
  纸带（paper_tape.parquet）
  ┌─────────────────────────────────────────────────┐
  │ 每笔完成交易:                                   │
  │   - 市场指纹 [L0, 趋势, 量能, 宽度]             │
  │   - 参数快照 [threshold, position_size, ...]    │
  │   - 交易结果 (pnl_pct, hold_days, exit_reason)  │
  └─────────────────────────────────────────────────┘
            │
  每日开盘前 ▼
  ┌─────────────────────────────────────────────────┐
  │ ① 计算今天市场指纹                              │
  │ ② 纸带上找最近邻 N 笔交易                       │
  │ ③ PnL 自然分组归因 (PnL>0 vs PnL≤0)            │
  │ ④ 有区分度 → 切换为赚钱组的参数                  │
  │ ⑤ 无区分度 → 保持当前参数                        │
  └─────────────────────────────────────────────────┘
            │
    平仓时追加纸带
```

## 七、运行入口

| 用途 | 命令 |
|------|------|
| 全链路日报 | `python scripts/run_pipeline.py --date YYYY-MM-DD`（**推荐**，自动 ①更新数据→②刷新信号表→③预计算因子→④运行策略→⑤生成日报→⑥写入Obsidian） |
| 实盘运行 | `python scripts/run_live.py [YYYY-MM-DD]` |
| 全量回测 | `python scripts/run_sniper_backtest.py` |
| 策略对比 | `python scripts/compare_strategies.py` |
| 日报生成 | `python scripts/daily_report_html.py` |
| 数据更新 | `python scripts/weekly_update.py`（增量） |
| 预计算因子 | `python scripts/precompute_l2.py` |
| 信号表更新 | `python -c "from scripts.run_pipeline import refresh_signal_tables; refresh_signal_tables('YYYY-MM-DD')"` |
| 深度分析 | `python scripts/analyze_deep.py` |
| 打字机优化 | `python scripts/optimize_typed.py` |
| 状态归因 | `python scripts/optimize_state_attribution.py` |
| 打靶采样 | `python scripts/optimize_target.py` |

### 全链路 6 步骤顺序（不可交换）

```
[①/⑥] 更新数据源（weekly_update）→ 日线、指数、行业指数
[②/⑥] 刷新信号表（refresh_signal_tables）→ 3 张表增量补缺：
         industry_compare  从 daily_bars+行业成分股计算（无API依赖）
         hot_stocks        从 daily_bars 涨停检测计算
         northbound        北向资金 API 下载
[③/⑥] 预计算 L2 因子（precompute_all）→ 确保 precomputed 模式可用
[④/⑥] 运行策略（daily_run）→ L0-L4 评分+归因+交易+纸带追加
[⑤/⑥] 生成日报（generate_report）
[⑥/⑥] 写入 Obsidian
```

### 信号表更新策略

| 表 | 更新方式 | API 依赖 | 滞后控制 |
|:---|---------|:--------:|:--------:|
| `industry_compare` | 从 daily_bars+行业成分股缓存计算行业日涨跌/量能 | ❌ 无 | ✅ 0 滞后 |
| `hot_stocks` | 从 daily_bars 按涨停规则检测 | ❌ 无 | ✅ 0 滞后 |
| `northbound` | akshare 东方财富 API | ✅ ~2s | ✅ 0 滞后 |
| `fund_flow` | 管道内跳过（独立脚本 download_all.py） | ✅ | P2 |
| `dragon_tiger` | 管道内跳过（独立脚本下载） | ✅ | P2 |

## 九、开发纪律

- **不可变数据类**：所有配置参数使用 `frozen=True` dataclass
- **单一真相源**：设计文档与代码必须同步（24 小时内）
- **数据操作必须先确认**：`DELETE`/`UPDATE`/文件删除需授权
- **代码变更自动提交**：每次任务完成 → git status → git commit
