# 量化系统 — 架构与日报归档

归档日期：2026-05-04

---

## 一、系统架构总览

```
量化日报.cmd (入口)
  ├── update_data.py  (数据层：增量更新)
  └── run_strategy.py  (应用层：生成日报 → Obsidian)
```

**三层漏斗模型：**
```
L1 大盘环境 (gate/layer1_market.py)  → 周线/月线MACD评估
L2 行业动量 (gate/layer2_sector.py)   → ETF分类指数评估
L3 个股评分 (gate/layer3_stock.py)    → 个股底部共振 + 顶部信号评估
```

**数据流：**
```
akshare → data/sources/ (DataSource抽象) → parquet缓存 → 回测/日报
```

## 六、L3评分修复记录（2026-05-04）

### 问题
原评分逻辑仅对 `buy_signal=True`（形态+放量）的"强势"股票赋值，其余通过门卫的"一般"股票 score=0，导致 Top N 排名和回测选股均随机。

### 修复
`gate/layer3_stock.py` 中 "一般" 分类评分改为：
```
score = (bullish_score * 5)     // 看多条件 0-20分
      + fund_conf * 20           // 资金信心 0-20分
      + mom_bonus                // 月涨幅 0-20分
      + range_pos * 25           // 月内位置 0-25分
      + min(chan_buy_score, 10)  // 缠论买点 0-10分
```

### 重算
- 仅重算最新月份（2026-04），耗时约72秒
- 通过率：3169/4318
- 分数分布：0 ~ 84（均值42.6），连续可区分

## 七、回测参数

- **回测区间：** 2019-01-01 至 2026-04-30
- **选股数：** Top 10
- **交易成本：** 双边万分之一（0.01%）
- **交易模式：** 买入持有至卖出信号（非每周清仓重开）
- **L1通过：** 持有多头仓位
- **L2通过：** 仅在强势行业ETF中选股
- **L3通过：** 只有月度评分 Top N 且通过门卫的股票可持仓
- **卖出信号：** L1/L2不通过（全清）或 L3跌出 Top N（单票卖出）
