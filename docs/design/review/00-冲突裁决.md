## 第二部分：P0 阻断项详细裁决（必须修复）

### 裁决 1：P-CRIT-01 — quotes 不支持北交所

**来源：** 通信协议院士 · 源代码验证（tdxpy/hq.py:88-94）
**严重度：** ⚠️ P0 阻断

**问题：** `get_security_quotes` 在检测到北交所 market=2 时返回 None，整批失败。

**裁决：** 在 `MootdxRealtimeSource.fetch_quotes()` 入口处过滤 `symbol.startswith(("8", "920"))`

---

### 裁决 2：A-CRIT-01 — 并发模型未定义

**来源：** 系统架构院士
**严重度：** ⚠️ P0 阻断

**问题：** 方案描述的 "ThreadPoolExecutor 保持流式" 没有给出具体结构。如果单线程串行，5000 只 × 0.5s = 41 分钟，不可接受。

**裁决：** N 个 worker 线程，每个内部串行闭环，worker 之间并发。
```python
def run(self, symbols):
    batches = self._chunk(symbols, self.n_workers)
    with ThreadPoolExecutor(max_workers=self.n_workers) as pool:
        for batch in batches:
            pool.submit(self._run_batch, batch)

def _run_batch(self, batch):
    for sym in batch:   # 串行闭环
        self._probe(sym) → self._validate(sym) → self._fetch(sym) ...
```

### 裁决 3：A-CRIT-02 — PROBE 依赖 FETCH 而非 WRITE

**来源：** 系统架构院士
**严重度：** ⚠️ P0 阻断

**问题：** Journal 中 FETCH=ok 但 WRITE 失败（断电），PROBE 误判为已完成。

**裁决：** PROBE 决策规则强制改为：
```
need_fetch = NOT EXISTS(step="write" AND status="ok" AND data_end >= today)
```

### 裁决 4：Q-CRIT-01 — 1% 采样统计效力不足

**来源：** 量化数据院士
**严重度：** ⚠️ P0 阻断

**问题：** 5000 只 × 1% = 50 只，1 只异常的发现概率仅 ~1%。

**裁决：** 双重阈值采样
```python
base_n = min(200, max(50, int(total * 0.02)))
# 动态调频：连续10只PASS → 降半；1只FAIL → 翻倍
```

### 裁决 5：Q-CRIT-02 — 跨源验证未处理复权差异

**来源：** 量化数据院士
**严重度：** ⚠️ P0 阻断

**问题：** mootdx quotes（不复权） vs akshare（前复权），除权日附近差异可达 20%+。

**裁决：** 跨源验证时，只取**非除权日前后 5 个交易日**的数据做 Pearson 对比。

### 裁决 6：Q-CRIT-04 — volume 单位差异 100 倍

**来源：** 量化数据院士。**已由协议院士的探针实测确认。**
**严重度：** ⚠️ P0 阻断

**实测数据：**
```
quotes('000001') → vol = 1083999（单位：股）
系统 daily_bars.volume → 单位：手（÷100）
写入后体积差 100×，volume-based 因子盘中盘后不一致
```

**裁决：** `MootdxRealtimeSource.fetch_quotes()` 输出层：`df["volume"] = df["vol"] / 100.0`

### 裁决 7：S-CRIT-01 — IP 封禁根因完全未处理

**来源：** 安全可靠性院士。**对照 2026-06-23 baostock 事故。**
**严重度：** ⚠️ P0 阻断

**问题：** mootdx 同 baostock 一样使用公共服务器，同样的高频全量扫描模式会导致同样的 IP 封禁。方案完全没有提及频率控制。

**裁决：**
```
1. 12:00 盘中：quotes() 调用 ≤ 1次/10秒（全市场 5000 只 = 5 次 TCP = 50 秒）
2. IP 轮换：从 consts.py 的 30+ 服务器随机选择
3. 心跳：heartbeat=True（默认 False！）
4. 熔断：3 次连续失败 → HealthTracker 标记不可用 → 降级 Tencent
5. 严禁：mootdx 用于 16:00 全量 5000 只并发日线下载
```

### 裁决 9：R-CRIT-03 — 首次迁移路径未设计

**来源：** 运维可观测性院士
**严重度：** ⚠️ P0 阻断

**问题：** 旧管道无 journal，首次切换全量重抓 40 分钟；新旧管道同时写入无协调。

**裁决：**
```
Phase 1: 部署新代码，创建 journal 表，旧管道继续，新管线只读 journal
Phase 2: 全量重抓 (skip_validation=True 加速)
Phase 3: data_end >= today-3 时切换
Phase 4: 删除旧锁文件
```

### 裁决 10：B-CRIT-05 — 跨股票数据源不一致（盲区）

**来源：** 盲区补充院士
**严重度：** ⚠️ P0 阻断（跨领域问题，协议+量化+架构交叉）

**问题：** 流式管线中股票 A 来自 mootdx（不复权），股票 B 来自 jqdata（前复权），跨股票因子比较（排序/百分位/配对）口径不一致，产生虚假信号。

**裁决：** 流式管线不改源选择策略——**所有股票使用降级链的第一个可用源**，保持全量一致性。降级是按源级别，不是按股票级别。
