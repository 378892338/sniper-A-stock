# 院士会审意见 #1 — 通信协议与数据格式

**审查专家：** 通信协议领域 · 二进制数据格式与 TCP 传输协议 院士
**审查范围：** 卷 A、卷 C.5、附录 1-2

---

## 1.1 肯定项

### [P1-A] TCP 7709 协议解析准确
方案对 `get_security_quotes` 的包结构、price varint 解码链、coefficient 乘数因子、K 线差值编码的逆向工程全部正确，与 `tdxpy/hq.py` 和 `tdxpy/parser/std/get_security_quotes.py` 源代码验证一致。

### [P1-B] 附录 1 的单位差异发现是关键
方案识别出 `quotes()` 的 `vol` 单位是 **股** 而系统的 `volume` 列习惯是 **手**（÷100），极可能避免了一个静默数据错误。这种来源端差异在测试中很难发现，只有在协议层对比才能暴露。

## 1.2 否决级问题

### 〖P-CRIT-01〗get_security_quotes 不支持北交所股票（8xxxxx/920xxx）

```
源代码验证（tdxpy/hq.py:88-94）:
    if isinstance(all_stock, list) and TDXParams.MARKET_BJ in [x[0] for x in all_stock]:
        warnings.warn("此接口暂时不支持北证A股", DeprecationWarning)
        return None
```

方案中 **12:00 盘中管道使用 `quotes()` 获取全部活跃股票**，但若包含北交所，整批返回 None。

**修复：** PROBE 阶段必须预先过滤北交所股票（symbol.startswith(("8", "920")) → 过滤）。

### 〖P-CRIT-02〗单次请求的最大股票数未验证

请求包容量 = stock_count × 7 + 12 bytes。TCP MTU=1500，超过约 210 只时会 IP 分片。通达信服务端可能对分片包有数量上限，方案中 "1000只/批" 的假设未经验证。

**修复：** batch_size 可配置，首次运行时自动二分搜索最优值（从 10/50/200/500/1000 递进探测）。

## 1.3 严重问题

### 〖P-CRIT-03〗servertime 解码公式不鲁棒

`_format_time` 假设 `reversed_bytes0` 永远是具体时间格式。实测发现通达信服务器在某些情况下返回 0 或异常值，导致乱码时间。

**修复：** servertime 仅做日志记录，不作为业务判断依据。

### 〖P-CRIT-04〗部分 ETF 的 coefficient 可能异常

A 股 coefficient=0.01 正常，但 SH_B_STOCK=0.001、SZ_INDEX=1.0。若 `get_security_coefficient` 对某些跨市场品种判断错误，价格差异可达 100 倍。

**修复：** 增加 price 交叉检查：`abs(price - last_close) / max(last_close, 0.01) < 0.5`。

## 1.4 建议汇总

| 编号 | 建议 | 严重度 |
|------|------|--------|
| P-CRIT-01 | 北交所股票过滤（8xxxxx/920xxx），不送 quotes() | **阻断** |
| P-CRIT-02 | batch_size 可配置 + 自动探测上限 | 高 |
| P-CRIT-03 | servertime 仅日志，不做业务判断 | 中 |
| P-CRIT-04 | price 与 last_close 交叉校验 | 高 |
| P-SUG-01 | vol 单位转换÷100 在源层完成 | 高 |
| P-SUG-02 | heartbeat=True 保活 | 中 |
