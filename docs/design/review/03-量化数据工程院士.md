# 院士会审意见 #3 — 量化数据工程

**审查专家：** 量化金融数据与因子工程 院士
**审查范围：** 卷 C.2-C.4、卷 D、卷 F

---

## 3.1 肯定项

### [Q1-A] 盘中盘后数据隔离正确
如果盘中不完整数据污染了 daily_bars，L2 因子会在 16:00 被静默修正但不可追踪。独立表的设计避免了此隐患。

### [Q1-B] Pearson 相关系数验证有效
0.995 阈值对 A 股合理——不同源因复权算法差异不可能完全一致，但 ≥0.995 保证趋势一致。

---

## 3.2 否决级问题

### 〖Q-CRIT-01〗1% 采样率在小样本下无统计效力

```
A 股 5000 只，1% = 50 只
1 只数据错误时发现概率 = 1 - (4999/5000)^50 ≈ 1%
→ 验证 100 次才发现 1 次
```

**修复：** 双重阈值采样：
```python
base_n = min(100, max(50, int(total * 0.02)))
# 动态调频：连续5只通过 → 降频；1只失败 → 翻倍
```

### 〖Q-CRIT-02〗未处理复权差异导致的跨源验证偏差

```
mootdx quotes() = 不复权实时价
akshare fetch_daily() = 默认前复权
除权日附近差异可达 20%+，Pearson 可能 < 0.995
```

**修复：** 跨源验证时：
1. 跳过除权日前后 5 个交易日
2. 或统一使用不复权数据比较

### 〖Q-CRIT-03〗12:00 的 close 不是最终收盘价，L2 因子不应使用

```
12:00 close = 上午 11:30 最后一笔
L2 因子应使用 15:00 收盘价
如果盘中 L2 使用实时 close = 盘中和盘后因子值不一致
```

**建议：** 12:00 管道中 L2 因子使用昨日 daily_bars.close + 今日实时变化率估算，而非 intraday_snapshot.close。

---

## 3.3 严重问题

### 〖Q-CRIT-04〗volume 单位不一致导致因子失效

```
quotes().vol → 股
get_k_data().vol → 手（与 daily_bars 对齐）
daily_bars.volume → 手

同一个"今日成交量"因子，盘中=30000股，盘后=300手
差异 100×，volume-based 因子盘中盘后完全不一致
```

**必须在 MootdxRealtimeSource.fetch_quotes() 输出层统一 ÷100 转换为手。**

### 〖Q-CRIT-05〗get_volume 解码 amount 的语义需验证

quotes 中 amount 来自 `struct.unpack("<I")` 后用 `get_volume()` 解码。但 `get_volume` 本来是成交量解码函数（带 logpoint 压缩），用于成交额可能不正确。

**建议：** 编写探针脚本输出 mootdx 和 akshare 同一只股票同一日的 amount 值做对比验证。

---

## 3.4 建议汇总

| 编号 | 建议 | 严重度 |
|------|------|--------|
| Q-CRIT-01 | 双重阈值采样（基础 n + 动态调频） | **阻断** |
| Q-CRIT-02 | 跨源验证跳过除权日区间 | **阻断** |
| Q-CRIT-03 | 12:00 管道不使用实时 close 做因子 | 高 |
| Q-CRIT-04 | volume ÷100 在源层完成 | **阻断** |
| Q-CRIT-05 | 探针脚本验证 amount 字段语义 | 高 |
| Q-SUG-01 | 连续 3 次验证失败触发熔断 | 高 |
