## 卷 A：基础研究 — 通达信数据格式全息图谱

### A.1 TCP 7709 二进制协议（底层基础）

通达信数据传输采用自定义二进制协议，非 REST/HTTP。所有请求走 TCP 7709 端口，一次连接可多次请求。

**连接生命周期：**
```
TCP connect(7709) → SetupCmd1 → SetupCmd2 → SetupCmd3 → [request/response]×N → close
                  │             │             │
                  ├─ 发送握手包   ├─ 发送配置   ├─ 初始化完成
                  └─ 验证服务端   └─ 适配参数   └─ 进入就绪状态
```

### A.2 实时行情协议 (get_security_quotes) — 12:00 管线的数据基础

**Request 包结构：**
```
Header(12B): 
  [0x10C(2B)][0x02006320(4B)][packet_len(2B)][packet_len(2B)][0x5053E(2B)]  // fixed
  [0(2B)][0(2B)][stock_count(2B)]                                              // dynamic
  
Body: stock_count × [market(1B)][code(6B)]
```

**Response 包结构（单只股票展开）：**
| 偏移 | 字段 | 类型 | 解码方式 | 物理含义 |
|------|------|------|---------|---------|
| 0 | `market` | B | 直接取值 | 0=深圳 1=上海 2=北京 |
| 1 | `code` | 6s | utf-8 decode | 股票代码，如 `000001` |
| 7 | `active1` | H | 直接取值 | 状态标记 |
| 9 | `price` | varint | get_price → ×coefficient | **当前成交价** |
| 9+N | `last_close_diff` | varint | (price+diff)×coeff | **昨收价** |
| 9+N | `open_diff` | varint | (price+diff)×coeff | **今开盘** |
| 9+N | `high_diff` | varint | (price+diff)×coeff | **最高价** |
| 9+N | `low_diff` | varint | (price+diff)×coeff | **最低价** |
| 9+N | `reversed_bytes0` | varint | → _format_time | **服务器时间** |
| 9+N | `reversed_bytes1` | varint | — | 单位权 |
| 9+N | `vol` | varint | get_volume | **总成交量（手）** |
| 9+N | `cur_vol` | varint | get_volume | **现量（当前笔成交量）** |
| 9+N | `amount` | I(4B) | get_volume | **总成交额（元）** |
| 9+N | `s_vol` | varint | — | **卖盘成交量** |
| 9+N | `b_vol` | varint | — | **买盘成交量** |
| 9+N | `bid1~5` | varint×5×2 | (price+diff)×coeff | 买一到买五价 |
| 9+N | `ask1~5` | varint×5×2 | (price+diff)×coeff | 卖一到卖五价 |
| 9+N | `bid_vol1~5` | varint×5×2 | — | 各档位挂买量 |
| 9+N | `ask_vol1~5` | varint×5×2 | — | 各档位挂卖量 |
| 9+N | `reversed_bytes4` | H | `(342,)` | 未知(保留) |
| 9+N | `reversed_bytes5~8` | varint×4 | — | 未知(保留) |
| 9+N | `reversed_bytes9` | h | /100.0 | **涨速** |
| 9+N | `active2` | H | — | 状态标记 |

**Price 解码算法（核心，CPU 密集型）：**
```
get_price(data, pos):
  1) 读取首字节，取低 6 位 → 初始值
  2) bit6 为符号位（0=正 1=负）
  3) bit7 为扩展标志：0=结束，1=继续读下一字节
  4) 每续一字节取低 7 位左移累加
  5) result = double(accumulated) × coefficient
     coefficient: A股=0.01, B股=0.001, 指数=1.0
```

**关键发现：** `price` 字段不是绝对价格，而是 **「当前价 × 1000」的整数**。`last_close`/`open`/`high`/`low` 都是差值（相对 price 的偏移）。解码公式：
```
真实价格 = (price + diff) × coefficient
```

### A.3 历史 K 线协议 (get_security_bars) — 日线数据的基础

**Response 单条 K 线结构：**
```
[year(2B)][month(1B)][day(1B)][hour(1B)][minute(1B)]
[open_diff(varint)][close_diff(varint)]
[high_diff(varint)][low_diff(varint)]
[vol_raw(I=4B)] → get_volume → 成交量(手)
[amount_raw(I=4B)] → get_volume → 成交额(元)
```

**价格解码（差值累积链）：**
```
open = (open_diff + pre_diff_base) / 1000
close = (price_open_diff + close_diff) / 1000
high  = (price_open_diff + high_diff) / 1000
low   = (price_open_diff + low_diff) / 1000
pre_diff_base = price_open_diff + close_diff  // 递推给下一条
```

**关键发现：** K 线价格使用 **差值编码**，每条 K 线的价格依赖前一条的 `pre_diff_base`。不是独立绝对值，必须顺序解码。

### A.4 本地 .day 文件格式（Reader 模块）

**文件路径：** `{vipdoc}/{sz,sh}/lday/{sz,sh}{code}.day`

**二进制记录格式：**
```c
struct day_record {
    uint32_t date;      // YYYYMMDD (如 20260625)
    uint32_t open;      // ×0.01 得实际价格
    uint32_t high;      // ×0.01
    uint32_t low;       // ×0.01
    uint32_t close;     // ×0.01
    float    amount;    // 成交额
    uint32_t volume;    // ×0.01 得实际成交量(股)
};
// sizeof = 32 bytes per record
// 整文件 = header(32B) + records×32B
```

### B.2 现有管线问题

|------|---------|

## 卷 C：架构设计 — 院士级方案

### C.0 核心原则

1. **协议级验证**：每个数据源的对接必须经过真实 TCP 包的 decode-decode 对等验证
2. **无假设设计**：不假设"xxx 应该返回什么"——每个对接点写探针实测
3. **审计第一性**：所有变更记录可追溯、可回放、可恢复
4. **流式原子性**：每只股票的 5 步闭环是一个原子事务

### C.1 总体架构图

```
┌─────────────────────────────────────────────────────────────────────┐
│                      Pipeline Orchestrator                          │
│  (run_pipeline.py — mode=intraday|full|recover)                     │
└───────────┬─────────────────────────────────────────┬───────────────┘
            │                                         │
    ┌───────▼───────┐                         ┌───────▼───────┐
    │  12:00 盘中    │                         │ 16:00 全量     │
    │  (mootdx TCP  │                         │ (Fetcher 链)   │
    │   实时快照)    │                         │               │
    └───────┬───────┘                         └───────┬───────┘
            │                                         │
            └──────────┬──────────────────────────────┘
                       │
              ┌────────▼────────┐
              │  Stream Engine  │  ← 核心变更
              │  (流式闭环引擎)  │
              └────────┬────────┘
```

### C.2 流式闭环引擎（Stream Engine）— 核心变更

```
═══════════════════════════════════════════════════════════════════
  每只股票一个独立原子事务：

  ┌──────┐   ┌──────┐   ┌──────┐   ┌──────┐   ┌──────┐
  │ PROBE │──▶│VALIDATE  │──▶│FETCH │──▶│WRITE │──▶│VERIFY │
  │ 探针  │   │验证数据  │   │抓取  │   │写入  │   │随机验 │
  └──────┘   └──────┘   └──────┘   └──────┘   └──────┘
      │          │          │          │          │
      ▼          ▼          ▼          ▼          ▼
  ┌────────────────────────────────────────────────────────┐
  │                  JOURNAL（审计日志）                     │
  │  每条记录: step|symbol|timestamp|status|detail(json)    │
  └────────────────────────────────────────────────────────┘
═══════════════════════════════════════════════════════════════════
```

#### C.2.1 第①步 — PROBE（探针）

**目的：** 判断该股票是否需要抓取，避免重复工作。

**逻辑：**
```
1. 从仓库读取该股票最新日期 → last_date
2. 从 Journal 读取该股票最新 fetch 记录 → last_fetch
3. 比对决策矩阵:

   | last_date | last_fetch     | 决策           |
   |-----------|----------------|----------------|
   | >= today  | 任何            | SKIP（已最新） |
   | < today   | 不存在          | FETCH（新股票）|
   | < today   | data_range 不含 gap | FETCH（增量）|
   | < today   | data_range 有 gap  | FETCH（补缺）|
   | 异常      | 失败            | RETRY（重试）  |

4. 写入 Journal: {step:"probe", status:"skip|fetch|retry", detail:{...}}
```

#### C.2.2 第②步 — VALIDATE（验证数据）

> **关键纠正：** 这里的"验证数据"不是验证源可用性（那是探针做的），而是验证 **从仓库读出的已有数据是否完整可信**。

**逻辑：**
```
1. 如果 PROBE 结果是 SKIP（已有最新数据）：
   → 验证仓库中该股票最后 N 条数据的完整性
   → 检查: close 无 NaN, volume 无异常零值, 日期连续无断层
   → 通过 = 跳过; 不通过 = 标记为 STALE 并进入 FETCH

2. 如果 PROBE 结果是 FETCH：
   → 验证数据范围：确认 start_date 到 today 之间有交易日
   → 验证交易日历：避免在非交易日耗费连接
   → 验证仓库 schema：目标列是否存在（ensure_schema 逐列检查）
   
3. 写入 Journal: {step:"validate", status:"pass|fail|stale", detail:{...}}
```

#### C.2.3 第③步 — FETCH（抓取）

**来源选择矩阵：**

| 管道模式 | 首选源 | 降级链 |
|---------|--------|--------|
| 12:00 盘中 | mootdx TCP `quotes()` | 腾讯 qt.gtimg.cn |
| 16:00 全量日线 | Fetcher 降级链 | jqdata→akshare_daily→...→mootdx k线 |
| 手动补缺 | Fetcher 降级链 | 同上 |

**mootdx TCP 实时行情抓取（12:00 专线）：**
```
fetch_realtime_quotes(symbols: list[str]) -> pd.DataFrame
  │
  ├─ 1. 分组: 沪市 batch(≤1000) + 深市 batch(≤1000)
  ├─ 2. 并发: ThreadPoolExecutor(max_workers=2)
  ├─ 3. 每批: client.quotes(symbols=batch)
  ├─ 4. 字段映射:
  │      code      → symbol
  │      price     → close
  │      open      → open
  │      high      → high
  │      low       → low
  │      vol       → volume (注意：quotes返回的vol单位是股，不是手)
  │      amount    → amount
  │      last_close → pre_close
  │      servertime → (记录到日志)
  ├─ 5. 防御校验:
  │      close 在 [0.01, 10000] 范围内
  │      volume >= 0
  │      open <= high, low <= close, etc.
  └─ 6. 写入 Journal: {step:"fetch", source:"mootdx_tcp", ...}
```

#### C.2.4 第④步 — WRITE（写入）

**写入目标决策：**

```
PROBE 结果 → 写入行为
─────────────────────────────────────
FETCH(增量)  → INSERT OR IGNORE（正常补缺）
FETCH(补缺)  → INSERT OR IGNORE（覆盖空洞）
RETRY        → 先 DELETE (symbol, date) 再 INSERT（纠正错误数据）
```

**12:00 写入特殊处理（重要）：**

```
盘中快照 → 写入 intraday_snapshot 表（独立表，不污染 daily_bars）
  CREATE TABLE IF NOT EXISTS intraday_snapshot (
      symbol    TEXT NOT NULL,
      date      TEXT NOT NULL,
      time      TEXT NOT NULL,        -- 服务器时间 HH:MM:SS.sss
      price     REAL,                 -- 现价
      open      REAL,
      high      REAL,
      low       REAL,
      pre_close REAL,
      volume    REAL,
      amount    REAL,
      bid1~5    REAL,                 -- 五档行情
      ask1~5    REAL,
      source    TEXT DEFAULT 'mootdx',
      PRIMARY KEY (symbol, date, time)
  );
  
16:00 全量 → 正常写入 daily_bars 表（不受盘中影响）
```

#### C.2.5 第⑤步 — VERIFY（随机验证）

**采样策略：**
```
验证比率 = min(1%, 50只/批次)  — 可配置
采样算法: Fisher-Yates shuffle 取前 N 只（保证无偏）
排除条件: 当日首日上市（无历史可比）
```

**验证内容：**
```
对每只采样股票:
  1. 另外选一个数据源（排除当前使用的主源）
  2. 获取同日期范围数据
  3. 比对 close 列的 Pearson 相关系数
  4. 阈值:
     │ corr ≥ 0.995  → PASS
     │ 0.98 ≤ corr < 0.995 → WARN（记录差异明细）
     │ corr < 0.98  → FAIL（标记为可疑，加锁保护）
  5. 写入 Journal: {step:"verify", status:"pass|warn|fail", detail:{...}}
```

### C.3 管线审计日志系统（Journal）— 全方案基石

#### C.3.1 日志格式

**物理存储：** SQLite 表 `pipeline_journal`

```sql
CREATE TABLE IF NOT EXISTS pipeline_journal (
    id         INTEGER PRIMARY KEY AUTOINCREMENT,
    run_id     TEXT NOT NULL,          -- 每次运行的UUID
    mode       TEXT NOT NULL,          -- 'intraday' | 'full' | 'recover'
    step       TEXT NOT NULL,          -- 'probe' | 'validate' | 'fetch' | 'write' | 'verify'
    symbol     TEXT NOT NULL,          -- 股票代码
    timestamp  TEXT NOT NULL,          -- ISO 8601
    status     TEXT NOT NULL,          -- 'ok' | 'skip' | 'fail' | 'warn'
    elapsed_ms INTEGER,               -- 耗时（毫秒）
    detail     TEXT,                   -- JSON 详情
    source     TEXT,                   -- 数据源名称
    data_start TEXT,                   -- 本次数据起始日期
    data_end   TEXT,                   -- 本次数据截止日期
    rows_count INTEGER,               -- 数据行数
    error_msg  TEXT,                   -- 失败时的错误信息
    UNIQUE(run_id, step, symbol)
);
```

#### C.3.2 日志读写 API

```python
class PipelineJournal:
    """管线审计日志 — 所有操作通过此接口，禁止直接写 SQL"""

    def __init__(self, db_path: str = "data/local/meta.db"):
        # 与现有仓库共用 SQLite 文件，不同表

    def start_run(self, mode: str) -> str:
        """开启一次新运行，返回 run_id (UUID)"""
        
    def log(self, run_id: str, step: str, symbol: str, status: str,
            detail: dict = None, source: str = None):
        """写入一条日志"""

    def get_last_run(self, mode: str, symbol: str) -> dict:
        """获取指定股票在指定模式下的最后一次运行记录
        
        Returns: {step, timestamp, status, data_end, ...}
        """

    def get_last_fetch(self, symbol: str) -> dict:
        """获取该股票最后一次成功的 fetch 记录
        
        用于 PROBE 阶段的决策依据
        """

    def get_missing_stocks(self, mode: str, symbols: list[str], 
                           today: str) -> list[str]:
        """获取需要更新的股票列表
        
        比对逻辑:
          1. 最近一次成功的 fetch 的 data_end < today
          2. 或者不存在成功的 fetch 记录
          3. 或者最近的 fetch 状态为 fail
        """

    def get_run_summary(self, run_id: str) -> dict:
        """统计某次运行的完成情况
        
        Returns: {total, ok, skip, fail, by_step: {...}}
        """

    def cleanup(self, keep_days: int = 30):
        """清理超过 keep_days 的旧日志"""
```

### C.5 mootdx 数据源改造

#### C.5.1 现有 `MootdxSource` 保持不动，新增两个子角色

```python
# data/sources/mootdx.py
```

## 卷 D：边界条件与防御设计

### D.1 mootdx TCP 连接异常处理

| 异常场景 | 检测方式 | 处理策略 |
|---------|---------|---------|
| TCP 连接超时 (15s) | `timeout` 参数 | 重试 1 次后降级到 Tencent 源 |
| 连接已断开 (closed) | `reconnect()` 检测 | 自动重连（内置心跳保活） |
| 数据波 GL (空响应) | `check_empty` 回调 | 重连 + 重试 1 次 |
| 服务器返回 | `ValidationException` | 跳过该股票，日志标记 `fail` |
| IPO 封禁 | HealthTracker 3 次连续失败 | 自动降级，15 分钟冷却期 |

### D.2 Journal 并发写入

```
多线程场景：每个 worker 线程独立写 journal
锁策略：使用 SQLite WAL 模式 + INSERT OR REPLACE
        线程级 batch commit（≤10 条/批）
```

### D.3 数据冲突矩阵

| 场景 | daily_bars | intraday_snapshot | 处理 |
|------|-----------|------------------|------|
| 12:00 写入 | 不操作 | 写入 | — |
| 16:00 写入 | INSERT OR IGNORE | 不操作 | — |
| 12:00 中断后 16:00 | 正常写入 | 残留数据 | 下次 12:00 覆盖 |
| 补缺（覆盖错误） | DELETE+INSERT | 不操作 | 事务包裹 |

### D.4 Journal 恢复场景

| 中断场景 | 恢复行为 |
|---------|---------|
| 管道在第 1000 只时崩溃 | 下次运行：PROBE 跳过已完成的 1000 只，从第 1001 只继续 |
| 单只股票 FETCH 失败 | 该股票标记 fail，重试阈值内重试，超阈值跳过 |
| Journal 表损坏 | `get_last_fetch` 失败 → 回退到仓库 `MAX(date)` 查询 |
| 整次运行中断 | 不清理旧 journal，下次 PROBE 时跳过 data_end >= today 的记录 |

## 附录 2：解码链路确认（从 TCP 字节到 DataFrame）

```
TCP 7709 Port → SetupCmd1/2/3
  → get_security_quotes([(0, "000001")])
    → 构建请求包: header(24B) + [market(1B) + code(6B)] = 31B
    → 接收响应包
    → parseResponse():
        → struct.unpack("<B6sH", buf[0:9])  → market, code, active1
        → get_price(body_buf, pos) ×N       → 每个价格一个可变长整数
        → struct.unpack("<I", buf[4B])      → amount 原始值
        → get_volume(amount_raw)            → 解压成交量
    → to_data(result_list)                  → pd.DataFrame
```

每只股票响应包约 135 字节。1000 只股票 ≈ 135KB 数据。TCP 7709 单次往返即可返回全部数据。
