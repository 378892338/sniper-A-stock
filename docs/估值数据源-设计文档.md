# 估值数据源 — 设计文档

> 补充 quant-system 缺失的估值层（PE/PB/市值/换手率）。腾讯 `qt.gtimg.cn` 全市场实时行情接口，按日追加快照。

**状态**: ACTIVE — 唯一真相源

---

## 设计约束

| 规则 | 应对 |
|------|------|
| 历史数据只能更新，不能删除 | INSERT OR IGNORE，只追加不覆盖，历史完好 |
| 不可篡改回测数据 | L2 factor 维持 `log(amount)` 不变，估值表从今天开始有数据 |

> **L2 不改的原因**：回测区间（2020-2024）没有历史估值值，改为真实市值反而缺数。

## 接口说明

### 腾讯行情接口

```
URL: https://qt.gtimg.cn/q=sh600519,sz000001,...
返回: GBK 编码，~ 分隔字段
频率: 单批 50 code，0.3s 间隔
```

字段索引（该接口固定格式）：

| 索引 | 字段 | 说明 |
|------|------|------|
| 1 | name | 股票简称 |
| 38 | turnover | 换手率 % |
| 39 | pe_ttm | 滚动市盈率 |
| 44 | float_mv | 流通市值（亿） |
| 45 | total_mv | 总市值（亿） |
| 46 | pb | 市净率 |

### 校验门（写入前）

6 道全部通过才允许写入：

```python
checks = [
    len(df) >= 3000,                         # 股票数达标
    df["total_mv"].notna().sum() >= 2000,    # 市值有效行达标
    df["total_mv"].isna().sum() / len(df) < 0.30,   # 市值缺失率
    (df["total_mv"] <= 0).sum() / len(df) < 0.20,   # 市值异常值率
    df["pe_ttm"].isna().sum() / len(df) < 0.50,     # PE 缺失率
    (df["pe_ttm"].fillna(0).abs() > 100000).sum() / len(df) < 0.01,  # PE 极端值率
]
```

校验失败 → 30s 后重抓一次 → 再失败则跳过当天。

### warehouse 接口

```python
get_valuation(date=None)          # → 最新日期快照（实盘用）
get_valuation(date="2026-06-11")  # → 指定日期快照
update_valuation(df) -> int       # → 追加快照，返回写入行数
```

### 写入策略

`INSERT OR IGNORE` 保证幂等——当天数据已存在时不会重复插入，也不会覆盖已有值。

## 数据流

```
run_pipeline.py:
  ①  weekly_update.py（K线+指数+日历）
  ①.5 update_valuation.py（try/except，失败不卡流程，记录返回值）
  ②  refresh_signal_tables（信号表）
  ③  precompute_l2（因子预计算）
  ④  daily_run（跑策略）
  ⑤  生成日报
```

## 注意事项

1. 接口只返回**当前值**，无历史值。系统从接入之日起按日追加。
2. `update_valuation.py` 可独立运行：`python scripts/update_valuation.py`
3. `__main__` 中才插入 `sys.path`，被 `run_pipeline.py` import 时不修改路径。
