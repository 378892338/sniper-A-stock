## 一、总览

### 一句话

**每笔交易记录市场指纹（多维度），新的一天来了，回头看纸带找最相似的交易归因，得出今天的最优参数。**

### 一句话总结

```
打字机打印 → 纸带不停生长 → 按需匹配 → 归因 → 参数切换 → 交易 → 打字机继续打印
```

## 二、打字机纸带

### 2.1 每笔交易记录什么

不是只记 L0 评分，而是记录完整的**市场指纹**：

```python
{
  # ── 交易的参数和结果 ──
  "params": {"stop_loss": -0.06, "position_size": 0.12, ...},
  "pnl_pct": 3.2,
  "hold_days": 15,
  "exit_reason": "profit_target",
  "symbol": "600519",
  "entry_date": "2026-03-15",
  "exit_date": "2026-04-02",

  # ── 市场指纹（多维度） ──
  "l0_score": 72,          # L0 合成评分（0-100）
  "l0_trend": 85,          # 趋势维度评分（0-100）
  "l0_volume": 65,         # 量能维度评分（0-100）
  "l0_breadth": 70,        # 宽度维度评分（0-100）
  "market_state_vector": [72, 85, 65, 70],  # 完整市场状态向量
}
```

**为什么是这么多维度？**

L0 合成的单一分数会丢失信息。举个例子：

```
场景 A: L0=63, 趋势=88（强势上涨）, 量能=45（缩量）
  → 实际是 "缩量上涨"，追高有风险

场景 B: L0=63, 趋势=42（弱势反弹）, 量能=88（放量）
  → 实际是 "放量反弹"，可能有反转

如果只看 L0=63，A 和 B 被当做相同市场状态
加了维度后，A 和 B 的距离很远 → 归因不会混用
```

### 2.2 纸带物理存储格式（parquet）

纸带使用 **parquet** 格式存储。由于 parquet 不支持深层嵌套 dict，存储时**展平**为扁平列：

**写入时展平规则**（`scripts/optimize_target.py` 和 `append_to_paper_tape()` 使用）：

```
原始记录:               →   parquet 列:
params.stop_loss        →   param_stop_loss
params.position_size    →   param_position_size
params.max_hold_days    →   param_max_hold_days
...
market_state_vector[0]  →   msv_l0
market_state_vector[1]  →   msv_trend
market_state_vector[2]  →   msv_volume
market_state_vector[3]  →   msv_breadth
其他标量                 →   保持原名（pnl_pct, entry_date, ...）
```

**读取时重建规则**（`load_paper_tape()` 使用）：

```python
# 从 parquet 读到 records 后，遍历重建
for r in records:
    r["params"] = {k.replace("param_", ""): r.pop(k)
                   for k in list(r.keys()) if k.startswith("param_")}
    r["market_state_vector"] = [
        r.pop("msv_l0"), r.pop("msv_trend"),
        r.pop("msv_volume"), r.pop("msv_breadth"),
    ]
```

**parquet 列清单（完整）**：

| 列名 | 类型 | 说明 |

| symbol | str | 股票代码 |
| entry_date | str | 入场日期 |
| exit_date | str | 出场日期 |

| l0_trend | float64 | 趋势维度 |
| l0_volume | float64 | 量能维度 |

### 2.3 纸带生命周期

```
初始化采样
  30 组随机参数跑回测 → paper_tape.parquet（~500笔）
  每笔交易带市场指纹向量

每天实盘
  收盘后 → 成功的交易追加到纸带（~1-3 笔/天）
  append_to_paper_tape(trade) → 展平写入 parquet

周末
  纸带多了 5-10 笔 → 重新归因检查
  如果有参数连续向同一方向漂移 → 标记市场演变趋势

1 个月后
  纸带多了 ~60 笔 → 归因基础更牢固

无限循环
  打字机不停 → 纸带不停生长 → 归因不停
```

**一条纸带记录一个完整交易决策周期**：什么参数 → 什么市场状态 → 赚了/亏了多少。每条记录回答一个问题——"在这种市场状态下，用这组参数交易会怎样？"

### 3.3 相似度计算

**归一化**：所有维度都是 0-100，天然统一尺度。

**距离函数**：加权欧氏距离

```python
def market_distance(fp1, fp2):
    """
    fp1, fp2 = [l0, trend, volume, breadth]
    """
    # 权重：L0 整体占 50%，趋势/量能/宽度共享 50%
    weights = [0.50, 0.20, 0.15, 0.15]
    diff = np.array(fp1) - np.array(fp2)
    weighted_dist = np.sqrt(np.sum(weights * diff ** 2))
    return weighted_dist
```

**找邻居**：

```python
def find_neighbors(today_fp, paper_tape, n=30):
    """
    在纸带上找距离最近的 N 笔交易。
    """
    distances = [
        (market_distance(today_fp, t["market_state_vector"]), t)
        for t in paper_tape
    ]
    distances.sort(key=lambda x: x[0])
    return [t for _, t in distances[:n]]
```

**邻居数量 N 怎么定？**

| 纸带总交易数 | 取 N 笔 | 说明 |
|------------|---------|------|
| < 50 | 全部 | 样本少，不挑 |
| 50~200 | min(40, total) | 逐步增多 |
| 200~500 | min(35, total) | 稳定区间 |
| > 500 | 30 | 足够多，不需要大幅增加 |

## 四、归因（PnL 自然分组 + 并行归因）

### 4.1 profit_impact — PnL 自然分组

不再用固定比例（70/30、50/50 都不行），改用 **PnL=0 作为天然分界线**：

```
对参数 k 的 N 笔近邻交易：

赚钱组 (PnL > 0)          ← 事实分组，不是比例
  参数值的中位数 = win_median

亏钱组 (PnL ≤ 0)          ← 事实分组，不是比例
  参数值的中位数 = lose_median

impact = win_median - lose_median
  → > 0：参数偏大有利
  → < 0：参数偏小有利
  → ≈ 0：无区分度
```

```python
def profit_impact(trades, param_k):
    vals = np.array([t["params"].get(param_k, 0) for t in trades], dtype=float)
    pnls = np.array([t["pnl_pct"] for t in trades], dtype=float)

    # 过滤 NaN PnL（数据异常、字段缺失、手动构造交易）
    valid = ~np.isnan(pnls)
    vals = vals[valid]
    pnls = pnls[valid]

    if len(vals) < 10:
        return {"impact": 0.0, "win_median": 0.0, "lose_median": 0.0}

    # PnL 自然分组：PnL>0 赚钱组，PnL≤0 亏钱组
    win_mask = pnls > 0
    lose_mask = ~win_mask

    # 任一组 < 3 笔 → 无区分度
    if win_mask.sum() < 3 or lose_mask.sum() < 3:
        return {"impact": 0.0, "win_median": 0.0, "lose_median": 0.0}

    win_median = float(np.median(vals[win_mask]))
    lose_median = float(np.median(vals[lose_mask]))
    impact = round(win_median - lose_median, 6)

    return {"impact": impact,
            "win_median": round(win_median, 4),
            "lose_median": round(lose_median, 4)}
```

**边界约定**：
| 条件 | 行为 |
|------|------|
| PnL=0 | 归入亏钱组（不赚钱就是亏） |
| 赚钱组或亏钱组为空 | 无区分度，impact=0 |
| win_median / lose_median snap 到 grid 后相等 | 无区分度 |
| PnL 为 NaN | 在 profit_impact() 开头过滤掉，不参与分组 |
| 过滤后任一组 < 3 笔 | impact=0 |

**删掉的字段**：
| 旧字段 | 删除原因 |
|--------|---------|
| `high_val` | 被 `win_median` 替代（赚钱组中位值，不是高百分位均值） |
| `low_val` | 被 `lose_median` 替代 |
| `n` | 无人使用 |
| `certainty` | 单位不统一（|impact| / pnl_std），无实际指导意义 |

### 4.2 并行归因（不过滤交易）

**核心洞察**：因为不过滤交易，每轮归因都是在全量数据上独立算 profit_impact()。固定参数 A 不影响参数 B 的计算结果 → **顺序不影响最终值**。

所以不需要"除以步长"做归一化——每个参数在自己的量纲下算 |impact| 即可。排序只是决定报告顺序，不影响每个参数最终取什么值。

```
对 params_meta 中每个参数 k，在全量交易上独立算 profit_impact()：
  impact_k, win_median_k, lose_median_k

先决条件：
  - 赚钱组或亏钱组为空 → impact=0，跳过
  - NaN 传播防御（win_median/lose_median 为 NaN → 跳过）
  - snap 到 grid 后 win_median == lose_median → 无区分度，跳过
  - |impact| ≈ 0（< 1e-8）→ 无区分度（不开枪原则 → 不输出）

对于有区分度的参数，取值 = win_median_k（赚钱组中位值，snap 到网格后）
对于无区分度的参数，不输出（B 方案：调用方保持当前值）
```

**B 方案输出规则**：

```python
# _attribution() 不再输出所有 7 个参数，只输出有信号的参数
# 无信号的参数不出现在返回的 dict 中
# 调用方用 .get() 访问，缺参数意味着保持当前值

today_params = _attribution(neighbors)
# 可能返回 {"stop_loss": -0.10, "position_size": 0.12}
# 如果全无信号 → 返回 {} → configure_for_today() 不更新任何参数

# 在调用方:
stop_loss = today_params.get("stop_loss", EXIT.stop_loss)  # 用当前值做 fallback
position_size = today_params.get("position_size", RISK.position_size)
```

**不开枪原则**：`|impact| ≈ 0` → 该参数不输出，直接用当前值。避免在噪音中浪费弹药。

### 4.3 纸带方向验证（独立于归因）

归因方向是否可靠？用独立验证交叉确认：

```
对每个有信号的参数 k，从纸带取交易：

按参数值中位数分两组：
  A 组：参数值 > 中位数（大值侧）
  B 组：参数值 ≤ 中位数（小值侧）

对比 A 和 B 的平均 PnL：
  impact > 0（偏大有利）→ 期望 A 均值 > B 均值 → 方向正确 ✅
  impact < 0（偏小有利）→ 期望 A 均值 < B 均值 → 方向正确 ✅
  否则 → 方向可能反了 ❌
```

验证方法（中位数分组 + 平均 PnL 比较）与归因方法（win_median − lose_median）不同，**不是自证循环**。

```python
def verify_attribution_direction(trades, attribution_result) -> dict:
    """纸带方向验证，每周调用一次。

    Returns:
        {k: {"direction": str, "verified": bool, "note": str}, ...}
    """
```

每周在 `scripts/weekly_check.py` 中调用，作为归因质量的独立监控。

## 五、自进化循环

### 5.1 周期流程

```
第1天 ── 初始化
  30 组随机参数跑回测 → paper_tape.parquet（~500笔）
  每笔交易带市场指纹向量

第2天 ── 第一个实盘日
  load_paper_tape() → 纸带就绪
  today_fp = [l0_score, l0_trend, l0_volume, l0_breadth]
  configure_for_today(...) → 归因出今天参数
  交易
  收盘后: append_to_paper_tape(today_trade) → 纸带追加 1 笔

第3+N天 ── 持续实盘
  纸带每天多 1-3 笔
  每天开盘前 configure_for_today()
  参数可能每天微调

每天收盘后 ── 追加交易到纸带
  if 有平仓交易:
    append_to_paper_tape(completed_trade)
  else:
    不做操作（无新增数据）

每周末 ── 自动归因检查
  本周多了 N 笔交易（N>=5 时才触发）
  不重新跑回测
  只是基于更多数据重新归因 + 打印"参数漂移报告"
  如果发现某个参数连续 2 周向同一方向漂移 → 标记市场演变

1 个月后 ── 纸带多了 ~60 笔
  归因基础更牢固，参数更稳定

3 个月后 ── 纸带多了 ~200 笔
  参数已经充分适应市场特性

无限循环 ── 打字机不停，归因不停
```

## 十、方案对比

| 对比项 | 分箱 V3（旧方案） | 打字机 V4（当前方案） |
|--------|----------------|--------------------|
| 分箱 | 预先切 10 箱 | **不分箱，按需取最近邻** |
| 边界突变 | L0=59 和 60 参数不同 | **无边界，平滑过渡** |
| 维度 | L0 一维 | **L0 + 趋势 + 量能 + 宽度 四维** |
| 匹配精度 | 粗（每箱 10 分跨度） | **精（每个 L0 值可不同）** |
| 纸带 | 用完就扔 | **持续保留，不断生长** |
| 自进化 | 静态 | **每周自动归因** |
| 纸带存储 | 无 | **parquet 展平存储** |
| 实盘入口 | 无 | **run_live.py** |
| 追加交易 | 无 | **append_to_paper_tape()** |
| 复杂度 | 低 | 中（多了一个距离计算 + 纸带管理） |
| 大道至简 | ✅ | ✅ ✅ |
| 归因算法 | 70/30 固定比例 | **PnL 自然分组** |
| 归因流程 | 减枝（逐轮过滤交易） | **并行归因 + B方案（不过滤）** |
| 验证 | 无 | **纸带方向验证（独立方法）** |
