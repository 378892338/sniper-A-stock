## 二、根因分析

### 问题1：牛市识别不灵敏（导致2019/2020/2024大幅跑输）

#### 根因1.1：L1看空拦截阈值过低

**位置**: `gate/layer1_market.py:24`

```
L1_BEARISH_INTERCEPT = 4
```

看空加权公式：周线顶背驰×3 + 周线死叉(近8周)×2 + 月线死叉/DIF<0×2

牛市恢复期内（如2019年初、2020年COVID后、2024年初），月线DIF刚从深负区回升，DIF<0持续6-12个月（贡献2分）。此时任何一次正常周线回调产生死叉（贡献2分），即可达到2+2=4分，刚好触发拦截。这是导致2020年全年L1判"偏弱"的直接原因。

#### 根因1.2：月线DIF<0被错误地作为看空信号

**位置**: `gate/layer1_market.py:111-112`

```python
monthly_dif_below_zero = bool((monthly_dif < 0).iloc[-1])
t3_monthly = monthly_death or monthly_dif_below_zero
```

DIF<0并不等于看空。牛市恢复期DIF从深负区回升到零轴上方通常需要6-12个月，这段时间DIF虽然为负但趋势向上，实际上是**看多信号**。只有DIF<0且正在下降才应被视为看空。

#### 根因1.3：牛市状态判定要求过高

**位置**: `gate/layer1_market.py:264-267`

```python
if strong_count == 3 and bottom_bullish >= 2:
    state = "牛市"
elif strong_count == 3:
    state = "震荡"
```

`bottom_bullish`要求市场`bullish_score>=2`。但在牛市恢复期，b3（周线底背驰）结构性不可能出现（价格做更高低点），b2（月线条件）因DIF<0常被扣分，导致`bullish_score`通常只有1。即使三市全强，也只能判"震荡"。

#### 根因1.4：创业板一票否决过于激进

**位置**: `gate/layer1_market.py:276-281`

```python
if chinext_bearish and state not in ("熊市", "偏弱"):
    state = "偏弱"
```

创业板波动率高，牛市中正常回调更容易触发MACD死叉。2020年创业板涨+64%但因其技术信号波动大，触发否决后将上证/深证两市强势一起压制到"偏弱"。

### 问题2：夏普比率低（全期0.226）

#### 根因2.1：震荡市仓位过高

**位置**: `gate/layer1_market.py:233`

```python
position_map = {"牛市": 1.0, "震荡": 0.70, "偏弱": 0.30, "熊市": 0.0}
```

震荡市占比52.7%（198周），仓位70%。无趋势市场中，每笔交易净负期望（交易成本0.3%），高仓位持续产生无效亏损。这是夏普低的主因。

#### 根因2.2：无个股止损

**位置**: `backtest/runner.py:559-581`

`_should_sell()`只有四个退出条件：L1熊市、L2无强势板块、ETF跌出强势池、L3评分不通过。缺少个股层面的价格止损和时间止损。持仓从100跌到70仍可持有，导致日胜率仅22.6%（大量浮亏日）。

#### 根因2.3：无最低评分门槛

**位置**: `backtest/runner.py:748-756`

买入逻辑贪婪地填满所有slot，不检查L3评分质量。偏弱市场中评分40-50的边缘股也被买入。

---

### 问题3：L2/L3门控方向与牛市需求相反（牛市跑输核心原因）

#### 根因3.1：L2门控阈值方向反了

**位置**: `gate/market_state.py:68-73`

```python
GATE_THRESHOLD_MAP: dict[MarketState, int] = {
    MarketState.BULL: 3,    # 最严格
    MarketState.VOLATILE: 2,
    MarketState.WEAK: 1,    # 最宽松
    MarketState.BEAR: 999,
}
```

当前设计：牛市要求3/5条件通过（最严），偏弱仅需1/5（最松）。设计意图是"牛市警惕回调，偏弱放宽参与"。但这套逻辑更适合**反转策略**（高估时收、低估时放），而本策略是多因子评分系统，牛市中板块普遍强势，收紧门控反而阻塞了候选池。

**趋势跟踪策略**的正确设计应相反：牛市快速上车（门放宽），偏弱时精挑细选（门收紧）。

#### 根因3.2：L2看空拦截阈值方向也反了

**位置**: `gate/market_state.py:61-66`

```python
BEARISH_THRESHOLD_MAP: dict[MarketState, int] = {
    MarketState.BULL: 3,     # 最易触发
    MarketState.VOLATILE: 4,
    MarketState.WEAK: 5,     # 最难触发
    MarketState.BEAR: 999,
}
```

牛市用最低阈值3分拦截看空信号，意味着"周线顶背驰(3)+日线死叉(1)=4分"就足以拦截。牛市正常回调期间的MACD信号频繁触发此阈值。

#### 根因3.3：L3看空加权固定值不区分市场

**位置**: `gate/layer3_stock.py:199`

```python
if bearish_weighted >= 5:
    all_pass = False
```

固定阈值5分不随L1状态调整。牛市中日线顶背驰（2分）+周线死叉（2分）=4分接近拦截，再加任意信号即被拦截。应随市场状态动态调整。

#### 根因3.4：退出信号无牛市阻尼

**位置**: `gate/layer3_stock.py:502-559`

六个退出信号在牛市和熊市中触发标准完全一致。周线MACD死叉在牛市中仅反映正常回调，却触发卖出。

## 五、风险提示

1. **阈值调整的双刃剑**：放宽拦截可能让熊市早期识别变慢。P0.6（BEARISH_THRESHOLD weak=3）意味着偏弱市场更易触发看空拦截，补偿了熊市识别。
2. **仓位降低**：震荡市50%虽提高夏普，但也降低了牛熊过渡期的收益上限。
3. **止损可能增加交易频率**：-15%止损在波动率高的A股可能频繁触发。
4. **建议分阶段验证**：先实施P0全部6项，运行快速回测验证，再决定是否实施P1。
