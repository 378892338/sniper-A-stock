## 一、解决什么问题

当前系统用**一套固定参数**打天下，但市场在变：

```
牛市到来时 → 需要重仓位、松停损、大止盈
熊市来袭时 → 需要轻仓位、紧停损、甚至空仓
震荡磨人时 → 需要严选股、短持有、见好就收
```

就算同一轮牛市，**上半场和下半场的最佳参数也不同。**

**需求：参数不是固定的，而是随市场状态动态调整、持续自进化。**

## 三、三阶段详解

### 第一阶段：初始化（一次性）

#### 做什么

30 组随机参数跑一次回测，收集所有交易。

```
回测范围: 2019-01-01 ~ 2026-06-03（7.5年，今晚先用2025~2026跑一轮）
并行进程: 3（兼顾本机承载）
总耗时: ~1小时（短区间）/ ~5-6小时（全量）
```

#### 产出

每笔交易是一条完整记录：

```json
{
  "trade_id": 1,
  "l0_score": 72,
  "l1_rank": 1,
  "l2_score": 85,
  "params": {"stop_loss": -0.06, "position_size": 0.12, ...},
  "pnl_pct": 3.2,
  "hold_days": 15,
  "exit_reason": "profit_target"
}
```

所有交易附上**入场当天 L0 评分**这张标签。**L0 评分不依赖策略参数**，同一个日期的 L0 是固定的。

---

### 第二阶段：打字机开始工作（纸带不断生长）

```
纸带初始状态: 528 笔交易（来自初始化回测）

每天实盘交易后:
  纸带.append(当天交易记录)
  
周末自动触发:
  for 每个 L0 分箱:
    重新归因（profit_impact）
    箱子参数自动微调
  输出 state_map_v2.json

下周末:
  交易更多了，归因更准了
  输出 state_map_v3.json

持续循环 → 参数随市场演变自进化
```

#### 分箱规则

| 项 | 值 | 说明 |
|----|----|------|
| 维度 | L0 评分 | 市场状态评分 0~100 |
| 分箱宽度 | 10 | 0~10, 10~20, ..., 90~100 |
| 箱子数 | 10 箱 | 连续评分离散化 |
| 不足 10 笔的箱子 | **单独成箱** | 不做归因，运行时自动借用最近有参数的箱子 |
| 归因样本要求 | ≥10 笔 | 样本太少不归因 |

#### 归因方法（profit_impact — 你确认过的公式）

```python
def profit_impact(trades, param_k):
    """
    公式: impact = avg_pnl(前30%高值) - avg_pnl(后30%低值)
    
    > 0  → 参数偏大赚钱 → 往大调
    < 0  → 参数偏小赚钱 → 往小调
    ≈ 0  → 不重要，跳过
    
    确定性 = |impact| / pnl_std
    """
```

#### 减枝流程（你确认过的）

```
第1步：该箱子里，7 个参数都算 profit_impact
第2步：按 |impact|（确定性）排序，最大的 = 主因
第3步：主因参数调整到赚钱侧的值，固定
第4步：从交易中过滤掉"主因参数对着亏钱方向"的交易（减小噪声）
第5步：剩余参数重新归因，找下一个主因
第6步：直到所有参数收敛或剩余 |impact| 都接近 0
```

### 第三阶段：运行时自动匹配（每天开盘前）

```python
def get_daily_params():
    """每天开盘前执行一次"""
    # 1. 算今天 L0 评分
    l0 = market_scorer.composite_score(today)
    
    # 2. 找到对应的箱子
    bin_key = int(l0 / 10) * 10   # L0=63 → 60分箱
    params = state_map.bins[bin_key].params
    
    # 3. 如果这个箱子参数为空（不足10笔）
    if params is None:
        params = find_nearest_bin(bin_key).params
    
    # 4. 全局切换参数
    configure_for_market(params)
```

#### 参数切换机制

```
改前（死引用 — 切换无效）:
  from sniper.config import EXIT as CFG
  CFG.stop_loss  → 永远是加载时的值

改后（活引用 — 切换生效）:
  import sniper.config as CFG
  CFG.EXIT.stop_loss  → 自动取最新的全局参数
```

## 四、被优化的参数

**第一轮优化 7 个参数**（受市场状态影响最明显的）：

| 参数 | 所属模块 | 为什么受市场影响 |
|------|---------|----------------|
| stop_loss | EXIT | 牛市松一点少被洗，熊市紧一点保命 |
| trailing_stop | EXIT | 牛市跟紧止盈，熊市不指望 |
| profit_target | EXIT | 牛市拿住大肉，熊市见好就收 |
| max_hold_days | EXIT | 牛市可以长持，熊市短线快跑 |
| position_size | RISK | 牛市重仓，熊市轻仓 |
| soft_min_score | ENTRY | 牛市可放宽入场，熊市严选 |
| bullish_threshold | MARKET | 牛市/熊市的分界点该设多少 |

**其他 30+ 参数不动**（保持 config.py 默认值）：

```
trend_weight=0.40, volume_weight=0.30, max_positions=5,
hard_min_price=3.0, commission_buy=0.00025, ...
```

这些不受市场状态直接影响，或者不是当前重点。**如果以后发现某个参数也跟市场状态相关，下一轮优化加进来即可。**

## 七、与旧版对比

### 对比 V1（决策树/Optuna）— 已废弃

| 对比项 | 旧方案 | 当前方案 |
|--------|--------|---------|
| 优化方法 | 决策树分裂 / Optuna TPE | 打字机 + 分箱归因 |
| 样本需求 | 大量（树分裂/贝叶斯） | 少量（10笔/箱即可） |
| 可解释性 | 低（黑箱） | 高（能说出"为啥选这个值"） |
| 计算成本 | 高（反复回测） | 低（一次回测 + 内存归因） |

### 对比 V2（逐轮固定参数）— 今晚之前写的

| 对比项 | 旧 V2 方案 | 当前 V4 方案 |
|--------|-----------|-------------|
| 回测次数 | 每轮30组×最多7轮=210组 | **1次30组** |
| 总耗时 | ~5-6 小时 | **~1 小时（短区间）** |
| 产出 | 一套万能参数 | **10 箱参数表** |
| 动态切换 | ❌ 不能 | ✅ L0 评分自动匹配 |
| 自进化 | ❌ 不能 | ✅ 周末自动归因更新 |

### 对比 V3（分箱归因—静态）— 我刚刚写的

| 对比项 | V3 分箱（静态） | V4 打字机归因（动态） |
|--------|---------------|-------------------|
| 纸带 | 用完就扔 | **持续保留，不断加长** |
| 更新机制 | 一次性产出 | **每周自动重新归因** |
| 智能优化 | ❌ 静态快照 | ✅ **自进化循环** |
| 打字机精神 | ❌ 违背 | ✅ **完全一致** |
