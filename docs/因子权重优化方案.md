# 因子权重优化方案

> 基于 4318 只股票 × 50 个月的 IC/ICIR/分组回测数据，用定量方法寻找最优因子权重组合。

---

## 一、背景

### 1.1 当前问题

系统现有 11 个因子，权重硬编码为 DEFAULT_WEIGHTS：

| 因子 | 当前权重 | 实际 IC | 方向 |
|------|---------|---------|------|
| trend_macd_5dim | 15 | -0.035 | **负向** |
| momentum_12_1 | 15 | -0.031 | **负向** |
| reversal_20 | 12 | +0.049 | 正向 |
| oversold_60 | 6 | -0.005 | 近零 |
| weekly_deviation | 4 | -0.044 | **负向** |
| bottom_divergence | 8 | +0.016 | 弱正向 |
| volume_reversal | 6 | — | 不稳定 |
| bottom_fractal | 4 | -0.009 | 近零 |
| low_volatility | 15 | +0.098 | **最强** |
| turnover_stability | 10 | — | 数据缺失 |
| close_position | 5 | -0.014 | 弱负向 |

两个最高权重因子（合计 30/100）IC 均为负值，意味着当前权重分配实际上在**奖励坏因子、惩罚好因子**。

### 1.2 设计目标

1. **数据驱动**：权重基于 IC/ICIR/分组单调性等定量指标计算，不靠主观判断
2. **动态调整**：系统自动根据最新 IC 数据校准权重，无需人工介入
3. **约束合理**：单因子权重上限 25，防止过集中；负 IC 因子不参与正权重分配
4. **与现有架构兼容**：不改三层漏斗架构，只改 L3 因子评分层的权重

## 二、权重优化算法

### 2.1 输入数据

| 数据 | 来源 | 说明 |
|------|------|------|
| 月频 Rank IC | `scripts/evaluate_factors.py` | 每月各因子 Spearman ρ |
| ICIR | 上述推导 | IC 均值 / IC 标准差 |
| 分组收益 | 上述推导 | Q1-Q5 分组平均收益 |
| 单调性评分 | 上述推导 | 5 分组中单调递增的比例 |

### 2.2 评分公式

每个因子 j 的综合有效性得分：

```
score_j = ICIR_norm_j × 0.50 + mono_norm_j × 0.30 + spread_norm_j × 0.20
```

其中：

- **ICIR_norm**：ICIR 经 min-max 归一化到 [0, 1]，负值置 0
- **mono_norm**：单调性评分（0-1），5 分组中 4 个相邻组按序递增的比例
- **spread_norm**：Q5-Q1 收益差归一化到 [0, 1]，负值置 0

**对负 IC 因子的处理**：IC_mean < 0 的因子 score 直接置 0（不参与正向权重分配）。

### 2.3 权重分配

```
w_j = 100 × score_j / Σscore_j
```

**约束条件**：
- `0 ≤ w_j ≤ 25`（单因子上限）
- `Σw_j ≈ 100`（归一化）
- 跳过 `turnover_stability`（数据缺失）和 `volume_reversal`（数据质量不足）

### 2.4 动态校准触发条件

| 触发条件 | 行为 | 频率 |
|---------|------|------|
| 月边界 | 自动运行优化，更新权重 | 每月 1 次 |
| L1 市场状态切换 | 即时触发校准 | 按需 |
| 手动调用 | CLI 执行 `--force` | 用户决定 |

## 三、校准与生效流程

### 3.1 校准管线

```
IC 数据采集 → 综合评分 → 权重分配 → 约束校验 → 写入内存 → 持久化
```

- **IC 数据采集**：利用已有 `evaluate_factors.py` 的输出 CSV
- **综合评分**：按 2.2 公式计算每个因子的有效性得分
- **权重分配**：按 2.3 公式计算权重，校验约束
- **写入内存**：覆盖 `multi_factor.py` 中的 `_last_calibration`
- **持久化**：保存到 `outputs/factor_eval/optimized_weights.json`

### 3.2 生效路径

```
get_dynamic_weights(force_recalibrate=True, market_state, store)
  → calibrate_weights()
    → optimize_from_ic_data()
      → 返回 CalibrationResult
  → 覆盖 DEFAULT_WEIGHTS 作用于当月评分
```

## 四、文件结构

| 文件 | 操作 | 说明 |
|------|------|------|
| `scripts/optimize_factors.py` | **新建** | 独立优化脚本，可直接运行输出推荐权重 |

| `factors/multi_factor.py` | **微调** | 确保 `aggregate_scores` 传入 `market_state` |
| `backtest/runner.py` | **微调** | 确保回测路径触发校准 |

## 五、验收标准

1. `scripts/optimize_factors.py --input outputs/factor_eval_4318` 输出推荐权重
2. 推荐权重中 `low_volatility` + `reversal_20` 合计占比 > 40%（当前两个最强因子）
3. `trend_macd_5dim` 和 `momentum_12_1` 权重合计 < 15（当前两个负 IC 因子）
4. 各因子权重在 [0, 25] 范围内
5. 权重总和 = 100（允许 ±0.1 浮点误差）
6. 回测路径自动触发月度校准
